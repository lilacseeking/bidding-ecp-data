# 提示词：电力物资需求预测期刊论文 — 全流程审查与方案制定

## 一、项目背景与目标

**最终目标**：撰写一篇可发表于中文核心或SCI期刊的研究生小论文（论文内容不需要你生成）。

**选题方向**：电力物资需求量预测。

**数据来源**：国家电网新一代电子商务平台 ECP2.0 (ecp.sgcc.com.cn)，已采集国网总部 2019-11~2026-07 共 81 个月数据，3 种目标物资（交流避雷器、电容式电压互感器、交流支柱绝缘子），含市场因子（铜/铝期货价格、工业用电量、农历编码）。

**当前模型**：`vmd-catboost/main.py` 中的 CatBoost、VMD-CatBoost、VMD-Transformer-CatBoost、VMD-Transformer、VMD-SVR。

**关键参考文档**：你需要在线搜索其他物资需求期刊做分析
- `bid-ecp-data/book/研究进展与问题解决方案.md`
- `bid-ecp-data/book/电力物资需求预测创新模型方案.md`
- `bid-ecp-data/book/国网投资计划影响因子可行性分析.md`
- `vmd-catboost/outputs/system_flowchart.md`

**项目代码位置**：
- `bidding-ecp-data/` — ECP 数据采集与处理
- `vmd-catboost/` — 预测模型

## 二、执行要求

1. **使用计划模式**：这是一个大型工程，需要先输出完整计划，你需要根据计划使用子agent做3轮审核，子agent需要提出问题，问题被解决后，子agent审核通过后再进行下一轮审核
2. **每个问题必须进行可行性验证**：不能仅做理论分析，需要搜索文献、查询数据、运行代码来验证结论
3. **用事实和数据说话**：有数据库的查数据库，有论文的引用论文，有代码的读代码验证
4. **不确定的事情标记为不确定**：不要猜测，明确标注哪些结论是确定的、哪些需要进一步验证
5. **输出文档**：所有分析结果写入 `book/` 目录下的 markdown 文件

## 四、需要逐一审查的10个问题

### 问题1：ECP 平台数据作为物资月度需求的合理性验证

**需要你做的事：**
1. 阅读 `bid-ecp-data/src/pipeline.py` 中的 Phase 4 解析逻辑，理解"货物清单"中的"数量"字段在业务上的含义
2. 查阅 ECP 平台的招标公告原文（随机抽 3-5 条），判断公告中 Excel 的"数量"列是"当月交付量"还是"整个项目总需求量"
3. 阅读 ECP 平台的招标批次预安排文档，分析批次节奏对月度需求数据的影响
4. 如果"数量"不是月度需求，而是项目总需求（如"一批次采购未来半年的量"），分析这对月度预测的影响以及如何处理
5. 搜索电力物资需求预测相关论文中使用的数据来源，对比我们的数据采集方式

**输出：** `book/数据合理性验证.md`
- 明确的结论：ECP 数据是否可以作为月度需求的代理变量
- 如果不能，提出数据变换方案（如从批次总量反推月度需求）
- 如果能，说明理由和局限性

### 问题2：冀北 vs 国网总部或其他物资的数据选择问题

**需要你做的事：**
1. 从数据库中提取冀北公司（如仍有旧数据）和国网总部数据的完整统计：
   - 每种物资的零值月数量、连续零值最长长度
   - 月度需求分布（均值、方差、偏度、峰度）
   - 是否有明显趋势或结构突变
2. 分析冀北数据中"零值过多"导致模型崩溃的具体机制是什么——是 VMD 崩溃、CatBoost 退化、还是训练样本过少？
3. 对于冀北数据，是否可以合并多种物资（如将冀北的所有物资需求汇总）来降低零值率？
4. 对于国网数据，36-38% 的零值率在电力物资预测文献中是否属于合理范围？
5. 给出明确的数据选择建议

**输出：** `book/数据选择分析.md`

### 问题3：期刊发表可行性审查 — 文献对比

**需要你做的事：**
1. 搜索近 3 年（2022-2025）电力物资需求预测相关的中英文期刊论文（至少 8-10 篇），提取每篇的：
   - 方法（ARIMA/LSTM/VMD/GRU/Transformer等）
   - 数据来源与规模
   - 评估指标（RMSE/MAE/MAPE/R²）
   - 核心结论
2. 将我们的方法与已发表文献对比，回答：
   - 我们的模型组合是否有已发表论文完全相同的情况？
   - 如果有，差异点在哪里？是否构成重复发表？
   - 我们的数据规模在同类论文中处于什么水平？
3. 检查我们当前使用的 4 种外部因子（铜价、铝价、工业用电、项目数量）在已有文献中是否被使用，是否有文献认为这些因子与物资需求无关
4. 检查"VMD + SVR"、"VMD + CatBoost"等组合是否已被发表，如果有，我们的做法有何不同

**输出：** `book/文献对比分析.md`

### 问题4：数据稀疏度问题 — 是否可以进一步合并

**需要你做的事：**
1. 从数据库中提取当前 3 种物资的零值分布热图（81 个月 × 3 种物资的 0/非0 矩阵）
2. 计算以下合并方案的零值率变化：
   - 方案A：只合并"交流避雷器"和"中性点避雷器"（同属避雷器大类）
   - 方案B：合并避雷器大类中的所有品类
   - 方案C：按国网批次维度聚合（如批次总需求量代替月度需求量）
3. 对每种方案，重新统计零值月数量和分布模式
4. 给出明确的结论：是否应该合并、合并到什么粒度
   
**输出：** `book/数据稀疏度与合并方案.md`

### 问题5：大类预测 vs 具体品类预测 — 需要基于数据裁决

**需要你做的事：**
注意：这个问题的背景是——我之前提出过"可以先预测大类总量，再按比例分配到具体物资"的方案（层次化预测），但你质疑了大类规律与具体品类规律的相关性。

1. 从数据库中按物资大类（避雷器类、互感器类、绝缘子类）汇总需求
2. 计算大类需求总量与具体品类需求量的 Spearman 相关系数，用数据回答"大类规律是否能代表具体品类规律"
3. 如果相关性确实低，给出解释（是什么业务原因导致同一大类下的不同品类需求模式不同？）
4. 如果相关性高，则验证层次化预测的可行性
5. 如果相关性低但你认为层次化预测仍有价值，从什么角度论证（如：大类的总量约束可以防止具体品类预测值超出合理范围）？

**输出：** `book/大类vs品类相关性分析.md`

### 问题7：影响因子相关性低的解决方案

**需要你做的事：**
当前 CatBoost baseline 的 R² 仅 0.15-0.49，说明现有的 7 个外部因子（项目数量、铜价、铝价、工业用电、工业用电同比、农历月 sin/cos）对物资需求的解释力不足。

1. 计算当前 7 个因子与 3 种物资需求的 Spearman 相关系数，按物资分别制表
2. 搜索电力物资需求预测文献中使用过的影响因子，整理成对比表
3. 结合 `book/国网投资计划影响因子可行性分析.md` 中列出的候选因子，对每个候选因子做可行性评估：

   - 采集成本（低/中/高）
   - 预期预测力（基于文献或业务逻辑）
4. 对以下新增因子逐一验证：
   - 批次事件特征（从 ECP 数据库直接提取）
   - 月度电网投资完成额（从 AKShare 获取）
   - 去年同期需求 YoY（已有 lag12，但可能有更好的表达形式）
   - 是否还有其他应该考虑的因子？
5. 给出明确的因子增强方案和预期效果

**输出：** `book/影响因子增强方案.md`

### 问题9：模型合理性评估

**需要你做的事：**
基于"电力物资需求预测"这个目标和"81 个月、36-38% 零值率"的数据特征，评估当前 5 个模型的合理性：

1. **CatBoost baseline**：
   - 树模型处理间歇性序列的适用性
   - 是否需要增加 zero-inflated 处理逻辑？

2. **VMD-CatBoost/Transformer/SVR**：
   - VMD 在大量零值序列上的数学性质——零值是否导致 VMD 的模态混叠？
   - 如果 VMD 不适合，是否有替代分解方法（如 STL 分解、Prophet 的趋势/季节/假日分解）？

3. **整体模型选择**：
   - 基于数据特征（中等长度、高零值率、外部因子解释力弱），应保留哪些、删除哪些、新增哪些？
   - 对比 `book/电力物资需求预测创新模型方案.md` 中的 7 个候选方案，哪几个最适合当前数据？
   - 如果没有找到完全合适的单个模型，是否可以在现有代码中做某个框架的改造达到目的？

4. **Baseline 选择**：
   - 论文中的 baseline 必须包含季节性朴素预测（Naive Seasonal）、持久性预测（Persistence）、SARIMA
   - 这些 baseline 的 R² 是多少？我们的模型是否显著优于它们？

**输出：** `book/模型合理性评估.md`

### 问题10：残差VMD方法的验证

在之前的讨论中，确定了正确架构是：`外部因子→Base Model→初步预测→残差→VMD分解残差→预测残差IMF→修正初预`。需要验证：

1. 对残差做 VMD 是否能提取出可预测的结构？
2. 残差 IMF 和外部因子的相关性是否比原始需求的 IMF 更高？
3. 给出完整的"残差 VMD"方法的伪代码和数学描述
4. 搜索文献中是否有类似方法（"two-stage prediction with residual decomposition"），避免重复

**输出：** 合并到 `book/模型合理性评估.md` 中

## 五、对答案的要求

1. **不准猜测**：不确定的事情不要用"可能""推测""应该"来包装成确定结论
2. **用数据说话**：能查数据库的必须查，能算相关系数的必须算
3. **标注置信度**：每个结论必须标注[确定]/[高置信]/[待验证]，让论文作者判断
4. **引用来源**：每个文献引用必须包含期刊名、年份、方法、核心指标
5. **保持怀疑态度**：对我的任何先前结论（包括"外部因子预测力低""VMD-SVR有数据泄露"等）都要重新审视验证，而非默认接受
