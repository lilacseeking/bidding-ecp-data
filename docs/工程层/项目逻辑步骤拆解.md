### S3: Excel解析入库 — 下载ZIP → 提取XLSX → 解析货物清单

| 属性 | 值 |
|------|-----|
| **目的** | 从公告附件中提取每笔物资需求的数量、单位、项目信息 |
| **方法** | 下载ZIP → 内存解压(XLSX识别) → `detect_column_map()` 自动列映射 → 逐行解析 |
| **输入** | 305条物资正刊公告的ZIP |
| **输出** | 366,101条 `bid_items` 记录 |
| **确定性** | 🟡 中 (50-80%) |
| **置信度** | [高置信] |

**发现问题：**

- **P-S3-1**: 🔴致命 | 列映射识别 `detect_column_map()` (pipeline.py:61-122) 只在第1行检测"物资名称"/"物料描述"签名。如果Excel表头在第2行(有合并单元格)，则完全无法识别。当前28条 no_file (pipeline日志)，比例约9%。
- **P-S3-2**: 🔴致命 | `demand_quantity` 的单位从未被归一化。`bid_items` 表中有 `unit` 列但仅用于统计日志，从未用于换算不同单位之间的数值。pipeline.py:620 直接将数值转float。`material_demand_total` 表跨单位做 `SUM(demand_quantity)` (demand_stats.py:91) 产生无意义数值(如5000米+10千米=5010，实际应为15000米)。
- **P-S3-3**: 🔴致命 | `demand_month` 使用公告发布日期 `notice_publish_time` (pipeline.py:453)，而非货物清单中的 `delivery_first`(首批交货日期)。公告发布的月份 ≠ 实际需要物资的月份。交货日期已在Excel中被解析但存入了 `remark` 字段(第601-612行)，未被利用。

### S4: 月度聚合 — 按物资+月份聚合需求量

| 属性 | 值 |
|------|-----|
| **目的** | 将物资明细按月度汇总，作为建模的目标变量 |
| **方法** | `material_demand_item`(按物资+单位+月份) → `material_demand_total`(跨单位聚合) |
| **输入** | bid_items表 |
| **输出** | material_demand_total: 1525种物资 × 81个月 |
| **确定性** | 🟡 中 (50-80%) |
| **置信度** | [高置信] |

**发现问题：**

- **P-S4-1**: 🔴致命 | 跨单位直接SUM (P-S3-2)，对目标三种物资(避雷器/互感器/绝缘子)影响较小但方法论有缺陷。
- **P-S4-2**: 🔴致命 | 零值月 = 该月没有发布包含该物资的公告 ≠ 该月需求为零。国网批次招标(一年4-6批)的公告集中在特定月份发布，中间月份没有公告，但需求可能在前后月份体现。`build_real_data.py:111-113` 对缺失月份直接填0。

### S5: 物资筛选 — 选择采购频率最高的物资

| 属性 | 值 |
|------|-----|
| **目的** | 从1525种物资中选出适合做预测的目标物资 |
| **方法** | 按 `COUNT(DISTINCT demand_month)` 降序，选前3名 |
| **输入** | material_demand_total表 |
| **输出** | 交流避雷器(52/81月)、电容式电压互感器(52/81月)、交流支柱绝缘子(50/81月) |
| **确定性** | 🟡 中 (50-80%) |
| **置信度** | [待验证] |

**发现问题：**

- **P-S5-1**: [待验证] | 3种物资在期刊论文中是"案例研究"规模。主流研究(Zhu et al. 2026)覆盖18类物资。3种可能被审稿人视为不够充分。需扩充到5-8种。
- **P-S5-2**: 🟡 | 即使"频率最高"的物资也有29-31个零值月(36-38%)。文献中同类数据零值率通常建议控制在30%以下。应评估合并物资大类是否能降低零值率。

## 模型训练流拆解

### S8: 特征工程 — preprocess_data()

| 属性 | 值 |
|------|-----|
| **目的** | 构造模型输入特征，分割训练/测试集 |
| **方法** | Spearman选top4因子 + lag1~12 + rolling_mean3/std3 + month_sin/cos → MinMaxScaler → split[:69]/[69:] |
| **输入** | df (81行 × 9列) |
| **输出** | X_train(69,13), y_train(69), X_test(12,13), y_test(12) |
| **确定性** | 🔴 低 (<50%) |
| **置信度** | [确定] |

**发现致命问题：**

- **P-S8-1**: 🔴致命 | **MinMaxScaler.fit(on ALL 81 months)** (main.py:483-484)。scaler的min/max参数从全部81月学习，测试集统计信息泄露到训练集。在浅层模型(如CatBoost)中影响有限，在SVR中可能影响拟合。不影响论文结论的定性方向，但审稿人会将其标记为技术错误。
- **P-S8-2**: 🔴致命 | **lag特征使用测试集未来值**。`np.roll(demand_raw, 1)` 在全部81个月上操作，然后 `[:69]`切分。lag1/lag2/lag3 在测试期引用了测试期相邻月份的真实值。在实际预测场景中，这些未来值不存在。评估结果不可信。
- **P-S8-3**: 🔴致命 | **rolling_mean3/std3 同样泄露**。测试期前3个月的滚动窗口包含了测试期数据。
- **P-S8-4**: 🟡 | **Spearman实际未执行**。`get_top_factors(material)` 在 main() 中首次调用无df参数，填充fallback缓存。preprocess_data中第二次调用命中缓存，返回固定前4个。因子选择实际没有用Spearman。
- **P-S8-5**: 🟡 | **农历编码未用于特征工程**。`lunar_month_sin/cos` 在数据中存在，但 feature 拼接只硬编码了公历 `month_sin/cos` (main.py:479)。农历信息未进入模型。

### S9: 模型一 — CatBoost Baseline

| 属性 | 值 |
|------|-----|
| **目的** | 提供不依赖VMD的纯外部因子预测基线 |

| **输出** | R²=0.15(避雷器)/0.49(互感器)/0.41(绝缘子) |
| **确定性** | 🟢 高 (>80%) |
| **置信度** | [确定] |

**发现问题：**

- **P-S9-1**: [确定] | R²=0.15-0.49说明外部因子预测力弱。这是有价值的发现。
- **P-S9-2**: 🟢 | CatBoost baseline 是最诚实的评估。没有VMD循环论证问题。

---

### S10: 模型二 — VMD-CatBoost

| 属性 | 值 |
|------|-----|
| **目的** | VMD分解需求序列 → IMF作为特征 → CatBoost预测 |
| **方法** | VMD(y_train)→IMFs_train + extrapolate→IMFs_test → [IMFs + factors] → CatBoost |
| **输入** | y_train, X_train_factors, X_test_factors |
| **输出** | R²=0.07(避雷器)/0.43(互感器)/0.48(绝缘子) |
| **确定性** | 🔴 低 (<50%) |
| **置信度** | [确定] |

**发现致命问题：**

- **P-S10-1**: 🔴致命 | **循环论证**。VMD分解 y_train 得到 IMFs，IMFs 作为特征去预测 y_train。由于 ΣIMF ≈ y，模型只需学习"加总"操作。用在论文中将被审稿人识破。
- **P-S10-2**: 🔴致命 | **测试期IMF = 去年同期IMF** (seasonal_naive)。模型预测 ≈ 去年同期的需求模式。不是真正的基于因子推理的预测。
- **P-S10-3**: [确定] | CatBoost的R²远低于SVR(0.07 vs 0.98)不是因为SVR更强，而是因为树模型的正则化(early_stopping/depth限制)阻止了它学"作弊"模式。

## 评估流拆解

### S14: 评估方法 — evaluate_model() + 单次分割

| 属性 | 值 |
|------|-----|
| **目的** | 评估模型预测精度 |
| **方法** | MSE/RMSE/MAE/R² on 单次 69/12 split |
| **输入** | y_pred_orig, y_test_orig |
| **输出** | 指标值 |
| **确定性** | 🔴 低 (<50%) |
| **置信度** | [确定] |

**发现问题：**

- **P-S14-1**: 🟡 | 单次分割(69/12)，测试集仅12个数据点。一个异常值可显著改变R²。没有rolling window交叉验证。
- **P-S14-2**: 🟡 | 缺少Diebold-Mariano检验(模型间差异的统计显著性)。
- **P-S14-3**: 🟡 | 缺少MASE(Mean Absolute Scaled Error)和sMAPE(处理零值月)。
- **P-S14-4**: 🟡 | 没有季节性朴素预测(Naive Seasonal)作为baseline。这是时序预测论文的标准基线。
