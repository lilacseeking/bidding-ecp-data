## 二、根本原因：这不是一个时间序列问题

### 2.1 核心发现：74 个月中只有 27 个批次事件

```
2020: 8月 (1种)
2021: 3月, 4月, 5月, 7月, 8月, 11月 (6次)
2022: 4月, 5月, 7月, 8月, 9月, 10月 (6次)
2023: 5月, 7月, 8月, 9月, 10月, 12月 (6次)
2024: 2月, 5月, 7月, 9月 (4次)
2025: 3月, 10月 (2次)
2026: 3月, 12月 (2次)
────────────────
合计: 27个批次事件 / 74个月 = 36%密度
```

这些并不是随机分布的零值。**零值代表"该月没有招标批次"。** 需求的发生不是每个月的独立伯努利试验，而是由招标批次这个离散事件驱动的。

### 2.2 为什么所有时间序列模型都失败

| 模型族 | 核心假设 | 在这个数据上成立吗 |
|------|---------|:--:|
| VMD | 信号在时间轴上连续变化 | **否**——信号只有 27 个非零点 |
| LSTM | 序列有马尔可夫性 (t 依赖于 t-1) | **否**——连续 5 个零值后突然 86,887，没有中间态 |
| CatBoost | 特征向量 X_t → 需求量 Y_t 存在统计映射 | **否**——|r| < 0.3 对全部 8 个外部因子 |
| Croston | 需求间隔和需求量都可用指数平滑预测 | **否**——间隔从 1 到 7 个月波动，指数平滑无法处理 |
| LightGBM Direct | lag1~lag18 + 滚动统计 + 外部因子 → 需求 | **否**——lag12 是唯一有效特征，其余 25 维都是噪声 |
| Naive-Seasonal | 今年某月 ≈ 去年同月 | **部分成立**——70% 的批次在相同季节出现 |

所有模型失败不是因为它们"不够好"，而是因为它们都在回答一个**错误的问题**。

**正确的问题：**
> 给定过去 6 年的 27 个招标批次，未来 12 个月内会发生几个批次？各自在哪些月？每个批次包含哪些物资？每种物资的量是多少？

**当前所有模型回答的问题：**
> 未来 12 个月的每个月，需求量是多少？

这是一个**离散事件预测**问题披上了时间序列外衣。任何在 74 个时间步上训练、输出 74 个连续值的模型，都在对真实的数据生成过程做了错误的假设。

## 三、正确的建模框架：批次事件预测

### 3.1 三层预测架构

```
问题分解:
  ┌─ When:  未来12月哪些月会有批次?
  │         历史规律: 平均间隔=2.6月, 集中月份=3-5月和8-10月, 每年4-6次
  │         方法: 历史频率统计 + 泊松过程
  │
  ├─ What:  该批次包含哪些物资?
  │         历史规律: 大品类(控制电缆+保护管+接线端子+绝缘子) → 几乎总是一起
  │                  通信单元 → 经常单独出现
  │         方法: 物资共现概率矩阵
  │
  └─ How Much: 每种物资需求量多少?
            历史规律: 同月不同年的量级相对稳定 (如 3 月均值 ~6万)
            方法: 季节性均值 + 年增速趋势
```

### 3.2 三层架构的具体实现

#### Layer 1: 批次时间预测 (When)

```
输入: 过去 27 个批次的月份
模型: 季节性泊松过程

P(第 i 个未来月有批次) = f(月份, 历史同月频率, 距上次批次的月数)

月份历史频率:
  1月: 0/6年 = 0%
  2月: 1/6年 = 17%
  3月: 4/6年 = 67% ★
  4月: 2/6年 = 33%
  5月: 4/6年 = 67% ★
  6月: 0/6年 = 0%
  7月: 4/6年 = 67% ★
  8月: 4/6年 = 67% ★
  9月: 4/6年 = 67% ★
  10月: 3/6年 = 50%
  11月: 1/6年 = 17%
  12月: 1/6年 = 17%

决策规则: 
  IF 月份 IN {3,5,7,8,9} THEN 大概率有批次
  IF 月份 IN {10} AND 年份<>2025 THEN 中等概率
  IF 距上次批次 > 3个月 THEN 提升所有概率
```

#### Layer 2: 物资选择 (What)

```
历史共现频率 (27个批次):

  全批次(5种): 2023-05, 2023-09, 2025-10 (3/27 = 11%)
  4种(不含通信单元): 2022-04, 2022-09, 2023-12, 2024-02, 2024-09, 
                     2025-03, 2026-03 (7/27 = 26%)
  4种(不含控制电缆): 0次
  3种: 5/27 = 19%
  1-2种: 12/27 = 44%

P(控制电缆 | 批次发生) = 出现在多少批次 / 总批次数
  = 15/27 = 56%

P(蝶式绝缘子 | 电缆接线端子也在批次中) = 同时出现 / 接线端子出现
  = 12/13 = 92%  ← 几乎总是一起买!

P(通信单元 | 全品类批次) = 出现在5种全批次中的概率
  = 3/7 = 43%  ← 通信单元只有不到一半的批次出现
```

#### Layer 3: 需求量预测 (How Much)

```
季节性均值 + 年增速:

3月需求均值 (历史同月):
  控制电缆: ~60,000 (2021-03, 2025-03, 2026-03)
  蝶式绝缘子: ~100,000 (2025-03, 2026-03)

9月需求均值:
  控制电缆: ~51,856 (2022-09, 2023-09)
  蝶式绝缘子: ~2,000 (2022-09, 2023-09, 2024-09)

年增速: 
  控制电缆 9月: 2022=51856, 2023=81440, 2024=35000 → 不单调, 均值 ~56,000
```

### 3.3 关键优势

| 维度 | 当前 ML 模型 | 批次事件模型 |
|------|:--|:--|
| 有效训练样本 | 74 个月 (80%为零) | 27 个批次事件 |
| 信号/噪声比 | <0.05 (外部因子噪声主导) | >0.8 (批次时间明确) |
| 可解释性 | 黑箱 | "3 月通常有一次 4 种物资的批次" |
| 训练需求 | 500+ 参数 | 0 参数 (纯统计) |
| 零值处理 | 模型自己学 | 天然 (没有批次 = 所有物资为零) |

## 四、与先前方案的交叉比较

| 方案 | 核心问题 | 为什么不够 |
|------|---------|-----------|
| 大类聚合 VMD | 大类间趋势不具可比性 | 已确认不可行 |
| TwoStage CatBoost | 分类+回归 P×Q 乘性放大 | 月均<30 的物资崩溃 |
| Croston+TSB+LGB | 2.6个月的平均间隔 vs 1-7月的实际波动 | 指数平滑假设被打破 |
| LGB Direct Multi-Step | lag12 是唯一有效特征 | 等同 Naive-Seasonal |
| Naive-Seasonal | 预测存在但需求量为零的月份 | 假阳性问题 |

**批次事件模型回答了一个不同的问题。** 它不是预测"12 个月的月度需求量"，而是预测"3-5 个批次，各包含 2-5 种物资"。数据不再是 (74,5) 的零膨胀矩阵，而是 (27, 3) 的批次表。

## 五、论文中的呈现

### 5.1 核心叙事

> "传统时序预测方法（LSTM、VMD-CatBoost、Croston）将电力物资月度需求建模为连续时间序列，忽略了该数据的底层生成过程——招标批次为离散事件驱动。本文提出一种基于批次事件分解的预测框架：先识别历史招标批次的时空规律（27个批次/74个月），再分解批次时间预测、物资共现预测和需求量预测三个子问题，每个子问题使用适配其数据特征的专门方法。"

### 5.2 与已有方法的本质区别

| 方法 | 对数据生成过程的假设 | 本文 |
|------|-------------------|------|
| Croston | 独立同分布的间隔和数量 | 批次共现 + 季节性间隔 |
| TwoStage | 分类 + 回归独立 | 层次依赖 (时间→物资→数量) |
| M5 LightGBM | 序列自回归 | 离散事件序列 |

### 5.3 预期 R² 提升

以控制电缆为例（当前最优 R² = 0.63）：

```
Naive-Seasonal: R² = 0.63 (只猜对"3月有需求"，但猜错了很多零值月)
批次事件模型:    R² > 0.7 (猜对3月+9月+可能的其他月，且不会在其他月产生假阳性)
```
