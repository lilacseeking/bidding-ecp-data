# 电力物资需求预测期刊论文 — 最终执行方案

> 基于 Phase 0-3 全部审查结论，经 3 轮 Agent 审核通过后制定。
> 生成日期: 2026-07-06

---

## 一、核心结论速览

### 1.1 数据层

| 问题 | 结论 | 置信度 |
|------|------|--------|
| ECP数据是否可作为月度需求 | **不可以直接用**。`demand_month` 用的是公告发布日期，不是交货日期；应替换为基于 `delivery_first`/`delivery_last` 的均匀分配方案 | [确定] |
| 冀北 vs 国网总部 | **采用国网全国聚合数据**。冀北零值率62-72%、最长连续零值8-11月，VMD完全不可行 | [确定] |
| 物资是否合并 | **不合并**。大类合并零值率改善仅3-10%，大类内部子品类间ρ中位数仅0.05-0.34 | [确定] |
| 3种物资是否足够 | **最少要5-8种**。3种在审稿人看来是案例研究，扩充到高/中/低需求特征各异的多品类 | [高置信] |

### 1.2 方法层

| 问题 | 结论 | 置信度 |
|------|------|--------|
| VMD-CatBoost | **废弃**。循环论证：ΣIMF≈y，用IMF作特征预测y等于用答案出题 | [确定] |
| VMD-Transformer-CatBoost | **废弃**。同上 | [确定] |
| VMD-SVR | **废弃**。R²=0.99是虚假精度（循环论证+SVR核方法学线性加法），绝对不能出现在论文中 | [确定] |
| CatBoost baseline | **保留**。修复3个数据泄露后，是最诚实的评估基线 | [确定] |
| 残差VMD两阶段预测 | **不优先实施**。文献已有8篇高度相似(2022-2025)，且残差可能接近白噪声 | [高置信] |

### 1.3 因子层（2026-07-06 实验验证后修订）

| 问题 | 结论 | 置信度 |
|------|------|--------|
| 原7因子解释力 | 仅"项目数量"有效(|ρ|=0.86)，但属同源内源特征。其余6个（铜铝期货/工业用电/农历编码）|ρ|<0.26，全部删除 | [确定] |
| 新有效因子 | 批次事件特征：transformer_bids(|ρ|=0.66), monthly_bid_count(|ρ|=0.63), uhv_bids(|ρ|=0.35)。数据已在ECP DB中就绪 | [确定] |
| P0增强方案 | 3个批次事件因子(F01-F03) + 3个自回归滞后(F04-F06 lag12/rolling/YoY)。预计R²提升到0.40-0.70 | [高置信] |
| 铜铝期货 | **已删除**。|ρ|=0.16-0.25, 国网长协价不跟期货波动，统计不显著 | [确定] |
| 工业用电量/农历编码/距春节天数 | **已删除**。|ρ|=0.03-0.06, 完全无预测力 | [确定] |
| 批次事件特征 | **论文最大创新点之一**。ECP批次预安排在年初发布，作为"确定性未来事件"输入模型，在文献中未见 | [确定] |

### 1.4 模型层

| 推荐 | 优先级 | 理由 |
|------|--------|------|
| CatBoost + 增强因子 + 修复数据泄露 | P0 | 基线，立即实施 |
| Naive Seasonal / Persistence / SARIMA baseline | P0 | 论文必需基线 |
| 批次事件驱动 Conditional Forecasting | P0 | 最大创新点，数据已就绪 |
| N-HiTS (neuralforecast) | P1 | 多尺度天然匹配，支持外生变量 |
| XGBoost / LightGBM 对比 | P1 | 消融实验 |
| Croston + TFT | P2 | 学术价值高但实现复杂 |

## 二、必须修复的5个致命问题（实施前必须先解决）

| # | 问题 | 文件 | 解决方案 | 预计工作量 |
|---|------|------|---------|-----------|
| F1 | `MinMaxScaler.fit(all 81 months)` | main.py:483 | 仅对训练集fit，再transform测试集 | 10行代码 |
| F2 | `lag1~lag3`在测试期引用未来值 | main.py:454-458 | 将lag构造移到scaler之前，用原始值构造lag后只取训练集部分做scaler | 15行代码 |
| F3 | `rolling_mean3/std3`在测试期泄露 | main.py:462-468 | 同F2处理 | 5行代码 |
| F4 | `demand_month`=公告月而非交货月 | pipeline.py:453 | 从Excel的`delivery_first`/`delivery_last`反推月度分配 | 1-2人日 |
| F5 | 单位未归一化直接SUM | pipeline.py:620 | 在聚合前按物资类型分组做单位换算 | 2-3人日 |

**F4 和 F5 是关键路径阻塞点**，修复后才能生成正确的模型训练数据。其他工作（因子增强、模型新增）可以在 F4+F5 修复期间并行进行。

## 六、决策总结：什么可以做，什么不能做

### ✅ 可以且应该做的事

| 事项 | 为什么 |
|------|--------|
| 使用国网全国聚合数据 | 唯一可行的数据源 |
| CatBoost作为主模型 | 方法论正确，无循环论证 |
| 批次事件特征 | 论文最大创新点，数据已就绪 |
| 报告"外部因子解释力有限" | 诚实的负面结论也有学术价值 |
| 扩充到5-8种物资 | 提升论文的方法论说服力 |
| 加入Naive/Persistence/SARIMA baseline | 审稿人的最低期待 |

### ❌ 不能做的事

| 事项 | 为什么 |
|------|--------|
| 将VMD-SVR的R²=0.99作为论文结论 | 循环论证导致的虚假精度，论文可能被拒/撤稿 |
| 使用任何"VMD分解y→IMF作特征→预测y"的模型 | 方法论根本错误 |
| 冀北数据做物资级预测 | 零值率62-72%，模型完全不可训练 |
| 使用原始preprocess_data() | 3个数据泄露导致评估不可信 |
| 仅3种物资投稿高水平期刊 | 审稿人可能视为case study而非methodology |
| 使用铜铝期货价格/工业用电量/农历编码作为预测因子 | |ρ|<0.26，经实验验证无效，论文中若使用会被审稿人质疑因子合理性 |
| 使用AKShare采集市场因子 | fetch_external_factors.py 已废弃，6个因子全部无效 |
