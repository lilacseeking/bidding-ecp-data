### 1.1 CatBoost Baseline 的深入评估

**问题：树模型处理间歇性序列是否合适？**

**结论：CatBoost是当前数据条件下合理且偏保守的选择，不需替换，但建议增加对比实验。**

理由：

1. **树模型天然支持零值处理**。与基于正态假设的统计模型（SARIMA、指数平滑）不同，梯度提升树不做数据分布的参数假设。零值月（36-38%）只是特征空间中一个合法区域，CatBoost通过分段常数函数可以学习"某些条件下输出接近零"的模式。文献支持：So (2024, *Scandinavian Actuarial Journal*) 在零膨胀保险索赔数据上系统比较了CatBoost/XGBoost/LightGBM，结论是CatBoost在零膨胀场景下表现最优，原因是其Oblivious Tree的对称结构减少了对稀疏数据的过拟合、内置L2正则化(默认lambda=3)以及MVS(Minimal Variance Sampling)对不平衡梯度分布的处理。

2. **不需要显式的Zero-Inflated处理**。Zero-Inflated Poisson/负二项回归适用于广义线性模型框架（需要指定分布族），但对梯度提升树而言，树的非参数属性已经部分隐含了这种能力。如果要引入，可以通过两阶段框架（分类是否有需求 + 回归需求量）显式建模零值过程，这与影响因子增强方案中"两阶段模型"的方向一致。但是，使用自定义ZIP损失函数（如So 2024提出的ZIPB2）直接嵌入CatBoost需要自定义梯度和Hessian，实现成本较高，优先度低于两阶段框架。

3. **不需要替换为XGBoost/LightGBM，但建议做对比实验**：
   - CatBoost的Oblivious Tree（对称树）在81月小样本下比XGBoost/GBDT的逐叶子分裂更不易过拟合
   - LightGBM的逐叶子生长(leaf-wise)在小样本上容易产生不稳定预测
   - **建议**：论文中保留CatBoost为主baseline，在消融实验中增加XGBoost和LightGBM的对比（不替换，只作为补充对比），展示模型选择的稳健性

## 二、废弃的VMD系列的简要确认（不超过200字）

代码资产价值评估中对4个VMD模型（#12-#15）的"废弃"判定是正确的。根本原因不是实现缺陷或数据问题，而是**方法论层面的循环论证**：VMD分解y_train得到IMFs，再将IMFs作为特征输入模型预测y_train，由于VMD的数学性质保证Sigma(IMF)约等于y，模型只需学习"加总"即可获得伪预测。CatBoost因早停和深度限制学不好这个简单映射(R²=0.07)，而SVR的RBF核在GridSearch下完美学会了它(R²=0.98) -- 这正是缺陷的证据而非能力的体现。因此四个VMD模型均不应出现在论文中，但VMD基础组件（分解/外推/筛选/K优化函数）算法实现正确，可在修正后的方法学框架中复用。

### 3.2 最终推荐：方案7 + 方案5 + 方案1

#### 推荐一：方案7 -- 批次事件驱动的Conditional Forecasting（P0，立即可做）

**推荐理由**：
1. **实现成本最低**：不改模型架构，只在CatBoost的特征工程中增加7-8个批次事件特征（is_bid_batch_month、batch_type、days_since_last_batch、cumulative_batches_this_year等）。数据已在ECP DB的bid_notices表中（928条公告，含notice_publish_time、batch_info等字段）。
2. **学术创新价值最高**："确定性的未来事件作为预测条件"是全新的思路。ECP批次预安排在每年初公开，我们可以提前知道"3月有输变电第二次设备招标"，将这种确定性信息编码为特征输入任何模型。文献中（包括最新的VMD-LSTM、TFT、Informer文献）都没有以这种方式利用批次信息——它们最多将批次作为隐式的季节模式学习。
3. **预期效果最好**：当前Spearman分析显示project_count（项目数量）rho=0.86-0.90，是唯一有效因子但有"同源泄漏"嫌疑。批次事件特征提供了一个完全独立的外部视角来捕捉采购节奏，预期R²提升最显著（10-20pp）。
4. **方法论诚实**：无循环论证，无数据泄露（批次安排是确定性未来信息，训练时在train set内学习批次-需求映射即可）。

**论文中的定位**：核心创新贡献。"区别于现有文献将批次信息作为隐式季节性模式学习，本文提出批次事件驱动的条件预测范式——将可预知的批次安排编码为确定性特征输入模型，使模型能够在预测时利用招标预安排信息。"

#### 推荐二：方案5 -- N-HiTS（P1，短期可实现）

**推荐理由**：
1. **天然匹配多时间尺度特征**：电力物资需求受年尺度（投资计划）、季度尺度（批次招标分布）、月度尺度（具体采购执行）三层节奏驱动。N-HiTS的三层Stack（pooling ratio=12/4/1）恰好对应这三个时间尺度，无需手动特征工程。
2. **实现简单**：Nixtla的neuralforecast库直接支持N-HiTS，一行`NHITS(h=12, input_size=36, ...).fit(df).predict()`即可运行。代码量远低于VMD-Transformer体系。
3. **领域首创**：N-HiTS自2023年AAAI发表以来在零售、能源等领域广泛应用，但在电力物资采购预测中是零应用。
4. **处理间歇性有独特优势**：N-HiTS的层次化插值在不同时间尺度上的MaxPool操作天然平滑了零值造成的断裂，无需零膨胀建模。

**论文中的定位**：对比模型。"N-HiTS以层次化时间分辨率建模多尺度需求模式，其三层架构分别对应年度投资周期、季度批次节奏和月度执行波动，为间歇性序列提供了一种不依赖信号分解的多尺度解决方案。"

**与外生因子的结合**：
- 方案A：将外部因子（铜价/铝价/用电量/批次事件）作为额外的外生变量通道，通过N-HiTS的exogenous接口直接输入（Nixtla支持）
- 方案B：外部因子单独通过MLP编码后注入每个Stack

#### 推荐三：方案1 -- Croston分解 + TFT（P2，中远期学术贡献）

**推荐理由**：
1. **专业处理间歇性**：Croston方法（1972年提出）将需求序列分解为"需求间隔"和"需求量"两个独立序列，是间歇性需求预测的经典方法。在此之前只用于零售库存预测，从未应用于电力物资采购。
2. **TFT提供可解释性**：Temporal Fusion Transformer的变量选择网络(Variable Selection Networks)可以量化每个外部因子对需求的贡献，分位数预测提供采购决策的不确定性区间。这是CatBoost和N-HiTS无法提供的。
3. **多层次创新**：Croston分解（理论层创新）+ TFT架构（方法层创新）+ 电力物资采购场景（应用层创新），在三个层面都有贡献。
4. **与现有Transformer代码有概念兼容性**：虽然TFT的代码实现不同于现有的SingleFeatureTransformer/MultiFeatureTransformer，但Transformer的概念（注意力机制、位置编码）是一致的。

**论文中的定位**：核心方法。"针对电力物资需求的高间歇性（36-38%零值月），本文引入Croston分解将需求序列分解为需求间隔和需求量两个独立子序列，通过TFT的分位数预测和变量选择网络实现可解释且带不确定性量化的预测。"

**实现风险**：TFT调参复杂度高（需要PyTorch Forecasting库），81个月的数据量对TFT而言偏少（通常需要>200步）。如果实验效果不佳，可退回到方案5（N-HiTS）作为核心方法。

## 四、Baseline 清单

### 论文中必须包含的 Baseline

| Baseline | 必须性 | 理由 |
|----------|-------|------|
| **Naive Seasonal (季节性朴素预测)** | **必须** | y_hat_{t} = y_{t-12}。这是时序预测论文的**强制性基线**。如果模型连"去年同期"都打不赢，就没有存在的价值。审稿人会首先检查这一点。 |
| **Persistence (持久性预测)** | **必须** | y_hat_{t+1} = y_t。简单但重要的下界基线。对于有强趋势的序列，Persistence可能优于Naive Seasonal。 |
| **简单均值/中位数** | **推荐** | y_hat = mean(y_train)。判断模型是否至少优于"猜均值"。 |
| **SARIMA** | **必须** | 统计方法的代表性基线。处理季节性+趋势的经典方法。81个月数据足够拟合SARIMA(1,0,1)(1,0,1)[12]。注意：SARIMA对零值敏感，零值月可能导致MA项无法收敛。如果SARIMA效果极差（甚至不如Naive），反而证明需要机器学习方法。 |
| **CatBoost (当前baseline)** | **必须** | 机器学习方法的代表性基线。需要在修复数据泄露+增强因子后重跑。 |
| **XGBoost / LightGBM** | **推荐** | 与CatBoost的同家族对比，展示模型选择的稳健性。并非每个都单独建模，可以是：选最优的GBDT变体 + 在消融实验节简提"XGBoost/LightGBM表现略逊于CatBoost（因零值/小样本）"。 |
| **随机森林** | **可选** | 如果是小样本（81个月），随机森林可能由于Bootstrap重采样而优于单一决策树基线，但通常不如GBDT。可作为"更简单ML baseline"。 |
| **VMD-去噪 + CatBoost (正确的VMD用法)** | **推荐** | 与废弃的VMD循环论证版本区分：正确做法是用VMD对全序列去噪（分解后去除噪声IMF再重构），然后用去噪后的序列训练CatBoost。这不构成循环论证（因为VMD没有看到"未来y"并将其作为特征）。此baseline用于论证VMD作为信号预处理的价值。 |

### Baseline 优先级顺序

```
1. 简单基线 (必须先跑):
   ├── Naive Seasonal (y_hat_t = y_{t-12})      ← 强制性
   ├── Persistence (y_hat_{t+1} = y_t)          ← 强制性
   └── Simple Mean (y_hat = mean(y_train))      ← 推荐

2. 统计基线:
   ├── SARIMA(1,0,1)(1,0,1)[12]                ← 强制性
   └── Holt-Winters 指数平滑 (可选)             ← 如果SARIMA失败

3. 机器学习基线:
   ├── CatBoost (当前baseline, 修复泄露后)      ← 强制性
   ├── XGBoost (对比)                           ← 推荐
   └── LightGBM (对比)                          ← 推荐

4. 深度学习/方法学基线:
   └── VMD-去噪+CatBoost (正确用法)             ← 推荐（区分循环论证）
```

## 五、评估指标清单

### 5.1 当前已有指标

| 指标 | 公式 | 当前状态 | 评价 |
|------|------|---------|------|
| MSE | mean((y-y_hat)^2) | ✅ 已有 | 标准指标 |
| RMSE | sqrt(MSE) | ✅ 已有 | 量纲一致，直观 |
| MAE | mean(|y-y_hat|) | ✅ 已有 | 对异常值不敏感 |
| R² | 1 - SS_res/SS_tot | ✅ 已有 | 与方差比较。对零值月敏感：如果零值月占比高，baseline的R²可能被零值拉低（分母SS_tot变小），需要配合MAE解读 |

### 5.2 必须增加的指标

| 指标 | 必须性 | 理由 |
|------|-------|------|
| **sMAPE (symmetric MAPE)** | **必须** | sMAPE = 200 * mean(|y - y_hat| / (|y| + |y_hat|))。上下界[0,200]，对零值有定义（不像MAPE在y=0时无穷大）。处理36-38%零值月数据的**必备指标**。 |
| **MASE (Mean Absolute Scaled Error)** | **必须** | MASE = MAE / MAE_naive，其中MAE_naive是Naive Seasonal在训练集上的MAE。MASE < 1 意味着模型优于Naive Seasonal。**时序预测论文的标准指标**。与尺度无关，可以在不同物资间比较。 |
| **DM检验 (Diebold-Mariano Test)** | **必须** | 模型间预测精度差异的统计显著性检验。回答"模型A的RMSE比模型B低0.5，这个差异是显著的还是噪声？"当前评估只有单次69/12分割，没有统计推断。12个测试点的DM检验功效有限但必须报告，可以根据MSE的渐近正态性构造统计量。 |

### 5.3 推荐增加的指标

| 指标 | 推荐度 | 理由 |
|------|-------|------|
| **RMSSE (Root Mean Squared Scaled Error)** | 推荐 | MASE的平方版本，对较大误差更敏感（M5 Competition标准指标）。允许在不同物资间比较预测精度。 |
| **Pinball Loss / Winkler Score** | 推荐（如果做区间预测） | 评估分位数预测的校准质量。特别适用于TFT（方案1）和N-HiTS（方案5）的分位数预测输出。 |
| **MAPE（仅非零月）** | 可选 | 在非零月子集上计算MAPE。虽然整体MAPE因零值月无定义，但非零月MAPE提供百分比误差的直观感受。需要在论文中明确"非零月MAPE"的范围限制。 |
| **Prediction of Change in Direction (PCD)** | 可选 | 预测方向的正确率（上升/下降/持平）。对采购决策有实际意义——"下月需求是否增加"可能比精确数值更重要。 |

### 5.4 不推荐的指标

| 指标 | 不推荐理由 |
|------|-----------|
| MAPE (全量) | 零值月（36-38%）导致分母为零，MAPE未定义 |
| 仅R²作为核心指标 | R²对零值月的敏感性会导致误导性结论（零值月的baseline预测=猜均值，R²被低估；但模型预测零值月的正确性应被MAE而非R²衡量） |

### 5.5 推荐的核心评估协议

```
1. 每次实验运行 Rolling Window CV（窗口=60月训练，12月测试，滚动步长=3月）
   而非单次69/12分割。生成每个指标的均值 +/- 标准差。

2. 报告以下指标:
   ┌──────────┬─────────────────────────────────┐
   │ 点预测    │ RMSE, MAE, sMAPE, MASE, R²      │
   │ 统计推断  │ DM检验 p值（每个模型 vs Naive)   │
   │ 区间预测  │ Pinball Loss (如果模型支持)      │
   │ 方向预测  │ PCD (可选)                       │
   └──────────┴─────────────────────────────────┘

3. 指标表格式（参考M5 Competition）:
   | Model | RMSE | MAE | sMAPE | MASE | R² |
   | 模型A  | xxx  | xxx | xxx   | xxx  | xxx |
   每列标注均值 +/- std (来自Rolling CV的n次切分)
```

## 六、残差VMD方法的数学描述 + 文献查重结果 + 可行性判断

### 6.1 方法定义（用户提出的方案）

**预测流程**：
```
外部因子 → Base Model → 初步预测 y_hat_base → 残差 r = y - y_hat_base 
→ VMD分解残差 r → 预测残差IMF → 残差修正项 r_hat → 修正预测 y_final = y_hat_base + r_hat
```

### 6.2 完整数学描述（伪代码/公式）

```
算法: 残差VMD修正预测 (Residual-VMD Correction, RVC)

输入:
  X ∈ R^{T×F}: 外部因子矩阵 (T个时间步, F个因子)
  y ∈ R^T:     真实需求量序列
  T_train:     训练集长度
  K:           VMD分解模态数
  M_base:      基础预测模型 (如CatBoost)
  M_res:       残差预测模型 (如CatBoost/LSTM)

步骤:

  # === Stage 1: Base Model 初步预测 ===
  X_train ← X[0:T_train, :],  y_train ← y[0:T_train]
  X_test  ← X[T_train:T, :],  y_test  ← y[T_train:T]
  
  M_base.fit(X_train, y_train)
  y_hat_train ← M_base.predict(X_train)    # 训练期预测
  y_hat_test  ← M_base.predict(X_test)      # 测试期预测

  # === Stage 2: 残差计算 ===
  r_train ← y_train - y_hat_train           # 训练期残差 (T_train维)
  r_test  ← y_test  - y_hat_test            # 测试期残差 (真值，仅用于评估)
  
  # === Stage 3: VMD分解残差 ===
  # VMD(Signal, alpha, tau, K, DC, init, tol)
  # 将 r_train 分解为 K 个IMF和一个残余趋势项
  u, u_hat, omega ← VMD(r_train, alpha=2000, tau=0, K=K, DC=0, init=1, tol=1e-7)
  
  # 识别残差IMF (最低中心频率分量) 和 模态IMF
  residual_imf_idx ← argmin(omega[-1])
  modal_imf_indices ← {0, ..., K-1} \ {residual_imf_idx}
  
  # 相关性筛选: 剔除与原残差Pearson低于阈值的噪声IMF
  valid_imfs ← filter_by_correlation(u[modal_imf_indices], r_train, threshold=0.1)
  
  # === Stage 4: IMF外推至测试期 ===
  # 对每个有效IMF，从训练期线性/季节性外推至测试期
  IMFs_test ← zeros(T_test, len(valid_imfs))
  for each imf_k in valid_imfs:
      if imf_k is trend component:
          # 线性外推: r = a*t + b
          (a, b) ← polyfit(0..T_train-1, imf_k, degree=1)
          IMFs_test[:, k] ← a*(T_train..T-1) + b
      else:
          # 季节性naive: 复制去年同期值
          for i in range(T_test):
              src_idx ← T_train - 12 + (i % 12)
              IMFs_test[i, k] ← imf_k[src_idx]
  
  # === Stage 5: 残差修正项预测 ===
  # 使用与Stage 1相同或不同的模型预测残差修正项
  # 输入: 外部因子 + 原始IMFs特征
  M_res.fit(X_train, IMFs_train, r_train)  # or 仅用IMFs
  r_hat_test ← M_res.predict(X_test, IMFs_test)
  
  # === Stage 6: 最终修正预测 ===
  y_final ← y_hat_test + r_hat_test
  
输出:
  y_final: 修正后的最终预测
  y_hat_test: 基础预测 (用于消融实验对比)
  r_hat_test: 残差修正项 (量化修正贡献)
```

#### 6.3.3 查重判断

"残差VMD"作为一个**独立的方法学贡献**发表空间有限（因为范式已存在）。但它作为**多阶段方法中的一个环节**是可发表的——前提是有其他创新维度支撑（批次事件特征、间歇性处理的跨领域迁移等）。

### 6.4 可行性分析

#### 6.4.1 残差的VMD分解是否有意义？（残差可能接近白噪声）

**关键问题**：如果基础模型（CatBoost）的残差接近白噪声（无自相关、无结构），VMD分解将没有意义——它会把白噪声强行分解为"伪模态"，每个IMF都没有预测价值。

**判断标准**：
1. **Ljung-Box检验**：对残差序列做LB检验。如果残差在所有滞后上都不显著（p > 0.05），说明残差已经是白噪声，VMD分解无意义。
2. **偏自相关函数(PACF)**：如果PACF在滞后12/6/3附近有显著峰值，说明残差仍有可提取的季节/周期结构。
3. **残差的信噪比估计**：计算残差的方差与原始序列方差的比值。如果残差方差占比 > 50%，说明Base Model解释力差，残差中的"信号"可能多一些。

**基于当前数据的推断**：
- CatBoost baseline R² = 0.15（避雷器）、0.49（互感器）、0.41（绝缘子）
- 这意味着残差占原始方差的 85%/51%/59%
- 避雷器的残差占比最高（85%），可能存在尚未提取的季节/批次模式
- 互感器的残差占比最低（51%），残差中的结构较少

**实际做法**：在实现前必须先跑LB检验和PACF图，判断残差是否"值得分解"。

#### 6.4.2 残差IMF与外部因子的相关性是否会比原始需求的IMF更高？

**预期：不会更高，反而可能更低。**

原因：
1. 残差 = y - y_hat_base。而y_hat_base已经是外部因子的某种函数f(X)。残差本质上已经"扣除"了外部因子的线性可预测部分。
2. 如果外部因子与y的相关性本就不强（Spearman rho=0.04-0.29），那么残差中与外部因子相关的成分更少。
3. 残差IMF可能代表的是：(a) 批次公告的不规则效应（没有外部因子的对应关系），(b) 真正的随机波动，(c) Base Model未能捕捉的非线性交互。
4. VMD分解残差得到的IMF可能只捕捉到残差自回归结构（如季节残余），而非外部因子驱动的新信息。

**但如果**：(a) Base Model是纯线性模型，未能捕捉非线性关系；(b) 外部因子与需求的非线性关系在残差中得到保留；那么残差IMF与外部因子之间可能存在微弱但非零的相关性。这需要通过实验验证。

#### 6.4.3 如果残差主要是噪声，VMD分解效果差，有什么替代方案？

**替代方案1：不对残差做VMD，而是直接用一个简单模型拟合残差**

```
残差 → 简单AR模型（如AR(12)）→ 残差修正
```
或者：
```
残差 → LightGBM（仅用时间特征：month_sin/cos + lag）→ 残差修正
```
这相当于在说"Base Model漏掉的季节性模式用一个简单模型补救"。
**优点**：简单、不引入伪结构。**缺点**：如果残差就是噪声，AR也学不到东西。

**替代方案2：VMD对原始序列去噪（非残差），然后用去噪序列 + 外部因子做预测**

```
y_train → VMD分解 → 去除噪声IMF(低相关性) → 重构去噪序列 y_denoised 
→ Base Model(X, y_denoised) → 预测
```

这是VMD的**正确用法**：VMD作为信号预处理（去噪），而非作为特征生成器。不涉及循环论证。

**文献支持**：Huang et al. (2022), "Material Demand Forecasting Based on Improved Particle Swarm Algorithm and Extreme Learning Machine" 中VMD的用法就是去噪而非特征生成。

**替代方案3：两阶段模型（推荐）**

```
Stage 1: 分类器 → P(本月有需求 | 批次事件 + 季节)
Stage 2: 回归器 → 需求量 | 有需求 （仅非零月训练）
```

这个方案不需要VMD，直接处理零值问题。与方案7（批次事件驱动）天然兼容。

**替代方案4：直接多模型Ensemble（放弃残差分解）**

```
CatBoost预测 → 特征A,
N-HiTS预测 → 特征B,
SARIMA预测 → 特征C
  └→ 元学习器(Meta-Learner, 如Stacking/Lasso) → 最终预测
```

不需要VMD，通过多种模型的互补来降低残差。

### 6.5 残差VMD方法的综合判断

| 评估维度 | 判断 | 说明 |
|---------|------|------|
| **方法学新颖性** | 低 | 范式已在多篇文献中发表（风速/负荷/碳价/大坝） |
| **在当前数据上的可行性** | 中低 | 残差可能接近白噪声（需要LB检验验证）；外部因子弱进一步降低了残差的可预测性 |
| **与论文中其他创新的协同性** | 中 | 可作为"批次事件驱动预测"框架下的一个子模块 |
| **发表风险** | 中高 | 审稿人可能质疑：(1)如果残差是噪声为什么VMD有意义？(2)为什么不直接改进base model？ |
| **推荐实施顺序** | P2（最后） | 先完成批次事件特征+CatBoost增强（方案7）和N-HiTS（方案5），如果这些方案仍有MASE>1（不如Naive Seasonal），再考虑残差VMD作为辅助修正 |

### 6.6 如果仍要实施残差VMD的建议路径

如果坚持实施，建议按以下步骤逐步验证：

```
Step 1: 验证残差的可分解性 (0.5人日)
  - 对 CatBoost baseline 的残差进行 Ljung-Box 检验
  - 绘制残差的 PACF 和频谱图
  - 如果LB检验p<0.05且PACF有显著滞后 → 继续
  - 如果残差是白噪声 → 放弃VMD，转用替代方案1或4

Step 2: 对残差进行VMD分解并检查IMF质量 (0.5人日)
  - VMD分解残差，K用中心频率分离度自动确定
  - 检查每个IMF与原残差的Pearson相关性
  - 检查IMF是否具有季节性模式（频谱上看是否有12月/6月周期峰）
  - 如果 >50% 的IMF被相关性筛选剔除 → 残差的VMD分解回报率低

Step 3: 简单残差修正测试 (0.5人日)
  - 不做VMD，用AR(12)直接拟合残差并修正
  - 用修正后的预测计算 RMSE/sMAPE/MASE
  - 与Base Model直接预测的指标对比
  - 如果AR修正后MASE下降 > 5% → 残差有修正价值
  - 如果AR修正后效果不变或更差 → 残差无可提取信号

Step 4: 仅在Step 1-3全部通过后，才实施完整VMD残差修正 (2-3人日)
  - 文献定位：不声称是"首次提出残差VMD修正方法"
  - 而是将其作为"多阶段修正框架中的一环，借鉴了负荷预测中的误差修正范式"
  - 差异化点：间歇性序列 + 批次事件特征的残差分解（目前无文献覆盖）
```

## 七、总结

### 推荐论文的模型实验方案

```
实验1: Baseline 验证
  ├── Naive Seasonal (强制性)
  ├── Persistence (强制性)  
  ├── SARIMA (强制性)
  └── CatBoost-原始因子 (当前baseline, 修复数据泄露后)

实验2: 特征增强 ablation
  ├── CatBoost + P0因子 (lag12, 批次事件, 春节距离)
  ├── CatBoost + P0+P1因子
  └── 消融: 逐个剔除因子测贡献

实验3: 新模型对比
  ├── 方案7: CatBoost + 批次事件特征 (核心方案)
  ├── 方案5: N-HiTS + 外生变量 (对比方案)
  └── XGBoost/LightGBM (模型选择稳健性)

实验4: (可选, 如果实验1-3的MASE仍>1)
  ├── 残差修正: AR(12) on residuals
  └── 残差VMD修正 (如果残差通过LB检验)

实验5: (远期, 发表期刊时)
  └── 方案1: Croston + TFT
```

### 评估指标

```
核心: RMSE, MAE, sMAPE, MASE, R²
统计推断: DM检验 (vs Naive Seasonal)
可选: PCD, Pinball Loss
```

### 交叉验证

```
Rolling Window CV: 窗口=60月训练, 12月测试, 步长=3月
报告: 均值 +/- 标准差 (所有指标)
```

### 残差VMD结论

该方法在文献中**已有先例**（2022-2025年间至少有8篇相关文献使用同一范式），不适合作为独立的"新方法"发表。在当前数据（81月、36-38%零值率、外部因子弱）上的可行性存疑（残差可能接近白噪声，VMD分解缺乏实际意义）。不建议优先实施，应先验证批次事件特征（方案7）+ N-HiTS（方案5）的效果。如果这两个方案已经能推动MASE<1（即优于Naive Seasonal），则论文的核心贡献已经成立，残差VMD可作为论文的"进一步讨论"或"展望"部分提及而非作为核心方法。
