### 数据生成逻辑

### 发现问题

**B0-1: [致命] 4个因子中 3个是所有物资完全相同的全局变量**


**数据验证**:
```
transformer_bids:  all sheets are IDENTICAL
monthly_bid_count: all sheets are IDENTICAL
uhv_bids:          all sheets are IDENTICAL
project_count:     DIFFERS per material (但差异很小)
```

**影响推演**:
- 模型看到的 4 个因子中，3 个对 5 种物资完全一样
- 唯一提供物资区分度的 `project_count` 在大多数月份对 5 种物资也是相同的
- → **模型被迫主要依赖自回归特征 (lag1/lag12/roll3) 做预测**
- → **本质上是 fancy 自回归模型，不是真正的"外部因子预测"**

**B0-2: [中等] COLUMN_CN 仍包含已删除的 has_batch / digital_bits**

`main.py:166-167` 保留 `has_batch` 和 `digital_bits` 的映射，但 `FACTOR_NAMES` 已移除。虽不影响运行（列在 Excel 中存在但不会被选中），但代码不干净。

**B0-3: [中等] project_count 的 Spearman ρ=0.86~0.90，与需求高度相关**

但 project_count 和 demand 来自同一张数据库表，属于同源信息。论文中须披露此限制。

### 审计结果

**S2-1: get_top_factors Spearman 因子选择 — 先前Bug已修复**

之前 `main()` 中首次调用 `get_top_factors(material)` 无 df 参数 → 缓存填充 fallback → Spearman 从未执行。当前 `main()` 中已移除预调用，修复完成。

**S2-2: Spearman 在全部 81 个月上计算 — 轻微数据泄露 [低严重度]**

`get_top_factors(material, df)` 中的 df 是完整 81 行数据（含测试集）。Spearman 在训练+测试混合数据上计算相关性 → 测试集模式可能影响因子选择。但影响极小：Spearman 系数通常在全量数据和仅训练集上相近。

**建议**: 传入 `df.iloc[:train_len]`（仅训练集），完全消除泄露。

**S2-3: Scaler 仅在训练集 fit — [正确]**

`feature_scaler.fit_transform(X_train_raw)` 和 `y_train = demand_train.copy()` — 测试集只做 `transform`。无数据泄露。

**S2-4: lag/rolling 无未来泄露 — [正确]**

训练集 lag 使用 `seq[:-1]`/`seq[:-12]`，测试集 lag 从 `demand_raw[train_len-*]` 取历史值。已修复的 `roll3[...](max(0,idx-3):idx])` 正确。

**S2-5: y 不归一化 — [正确但有代价]**

当前 y 保留原始值（不归一化），避免了 MinMax 测试集溢出 Bug。代价：CatBoost 在处理大数值（0~7580）时梯度可能不稳定。树模型理论上对标度不变，但实际中 `learning_rate` 和 `l2_leaf_reg` 的有效性受 y 标度影响。

**S2-6: [关键] is_zero_lag12_tr 语义问题 — [轻微逻辑错误]**

```python
is_zero_lag12_tr = (lag12_tr == 0).astype(float)
```

`lag12_tr` 是从 demand_train 计算的滞后值。但 demand_train 的 `lag12[:12]=0` 是构造时的默认值（前12个月没有 lag12 值）。这导致前12个月全部被标记为 `is_zero_lag12=True`。正确的做法是区分"历史值是零"和"历史值不存在"：

```python
is_zero_lag12_tr = np.zeros(n)
for i in range(12, n):
    is_zero_lag12_tr[i] = (demand_train[i-12] == 0)
```

**影响**: 前12个月的 `is_zero_lag12` 特征存在系统性偏差。69个月中12个月受影响（17%）。测试集 lag12 值从 `demand_raw[train_len+i-12]` 计算，不受此 Bug 影响。

**S2-评估**: **条件通过**。1 个低严重度问题 (S2-2)，1 个轻微逻辑错误 (S2-6)。整体特征工程方向正确。

## S6: _two_stage_fit_predict() — 统一两阶段框架

### 现有逻辑

```python
def _two_stage_fit_predict(X_tr, y_tr, X_te, stage2_regressor):
    y_bin = (y_tr > 0).astype(int)
    n_pos, n_neg = y_bin.sum(), len(y_bin) - y_bin.sum()
    if n_pos < 5 or n_neg < 5 or (y_tr > 0).sum() < 10:
        return None, None  # 回退信号

    # Stage 1: 共享分类器
    cls = CatBoostClassifier(...)
    cls.fit(X_tr[:-nv], y_bin[:-nv], eval_set=(X_tr[-nv:], y_bin[-nv:]))
    prob = np.clip(cls.predict_proba(X_te)[:,1], 0, 1)

    # Stage 2: 传入的回归器(仅在非零数据上训练)
    nz = y_tr > 0
    stage2_regressor.fit(X_tr[nz][:-nv2], y_tr[nz][:-nv2], ...)
    return prob * np.maximum(stage2_regressor.predict(X_te), 0), cls
```

### 审计结果

**S6-1: 分类器和回归器共用一个共享分类器 — [正确，但每次调用创建新分类器浪费]**

每个模型调用 `_two_stage_fit_predict` 都重新创建、训练一个 CatBoostClassifier。CatBoost/CondCatBoost/LightGBM 三个模型各自训练独立分类器 → 训练时间 3×。

**S6-2: 回退逻辑正确 — [通过]**

n_pos<5 或 n_neg<5 时返回 None → 调用方回退到直接回归。

**S6-3: [关键] Stage2 仅在非零数据上训练 — 这是正确的设计但存在统计风险**

- SGCC: 42/69(61%) 个月非零 → Stage2 训练样本 ~36 个
- Jibei: 21-31/69(30-45%) → Stage2 训练样本 ~18-28 个
- 36个样本 × 8个特征的 CatBoostRegressor(depth=6) → 严重过拟合风险

**证据**: 在断路器保护(月均需求24，CV=1.25)上，两阶段 R²=0.35 → 退化明显。

**S6-4: [中等] nv2 = min(6, nz.sum()//4) — 验证集可能过小**

当 nz.sum()=42 时 nv2=6，Stage2 回归用 36 样本训练 + 6 样本验证。对于 depth=6 的树模型，验证集只有 6 个样本 → 早停不可靠。

**S6-评估**: **通过，有统计风险**。回退逻辑正确，但小样本下两阶段方法的固有局限应诚实讨论。

## S7-S9: 两阶段模型 (CatBoost/CondCatBoost/TwoStage)

### run_catboost (S7)

新实现通过 `_two_stage_fit_predict` 实现两阶段预测。与旧版（直接回归）的关键差异：
- 旧版: 直接 reg.fit(X, y) → predict
- 新版: cls.fit → prob × reg.fit(nonzero) → predict

**S7-1: [已审] 旧版函数已成为死代码**

L767-796 定义的旧版 `run_catboost` 被 L1353 的新版覆盖。Python 使用最后一个定义 → 新版生效。旧代码保留但永不执行。

**S7-2: [关键发现] 两阶段统一化对 R² 的提升**

旧版 CatBoost（直接回归，含零值）vs 新版（两阶段）:

| 物资 | 旧版 R² | 新版 R² | 提升 |
|------|:---:|:---:|:---:|
| 交流避雷器 | 0.09 | **0.58** | +544% |
| 电容式电压互感器 | 0.14 | **0.36** | +157% |
| 交流支柱绝缘子 | 0.30 | 0.15 | -50% |
| 断路器保护 | 0.62 | 0.31 | -50% |
| 电抗器保护 | 0.34 | 0.36 | +6% |

**解释**: 两阶段对大需求波动物资效果显著（zero-to-spike 模式被 Stage1 捕捉）。小需求量物资退化 → Stage2 在 ~36 样本上过拟合。

### run_conditional_catboost (S8)

与 S7 相同框架，仅 CatBoost 参数更强（depth=7, iterations=2000）。在大需求量物资上轻微劣于 CatBoost（过拟合），在小需求量物资上有时更优。

L1381-1440 的 `run_two_stage` 拥有自己的 Stage1/Stage2 逻辑，与 `_two_stage_fit_predict` 功能几乎相同。差异：
- TwoStage 分类器 iterations=800 (vs 框架的 600)

**S7-S9-评估**: **条件通过**。两阶段化方向正确，对有高方差的大需求物资改善显著。代码重复需重构。

## 核心问题总结

### A: 两阶段统一化的收益评估

| 指标 | 数值 |
|------|------|
| 大需求物资平均 R² 改善 | +157%~+544%（0.09→0.58） |
| 小需求物资平均 R² 变化 | -24%~-50%（退化） |
| 论文可用性 | 4/5 物资 TwoStage系列显著优于统计基线 |
| 根本局限 | Stage2 仅有 ~36 非零训练样本，回归器不稳定 |

**建设性方案**:
1. **Stage2 用更小的模型**: depth=4 代替 depth=6，减少过拟合
2. **Stage2 用 ridge regression**: 线性模型在 36 样本上比树模型更稳定
3. **Ensemble**: 对 CatBoost/CondCatBoost/LightGBM 的预测取中位数 → 降低单模型方差

### B: 因子系统根本缺陷

**4 个 FACTOR_NAMES 中 3 个完全相同**（transformer_bids / monthly_bid_count / uhv_bids 对所有物资一样）。

模型只能通过以下途径区分物资:

| 特征 | 物资区分度 |
|------|:---:|
| project_count | 低（大部分月份相同） |
| lag1 / lag12 / roll3 | 高（自回归，物资特有） |
| is_zero_lag1 / lag12 | 中（反映各物资零值模式） |
| 月份/季度编码 | 无（全局相同） |

**建设性方案**:
1. **短期**: 用 `np.log1p(demand)` 构造目标 → 降 CV 从 1.11 到 0.35
2. **中期**: 添加物资级衍生特征 (如: demand_yoy=lag1/lag12-1)
3. **长期**: 从 ECP 公告正文提取真正的物资级外部信号 (如中标金额、供应商数量)

### C: 未被发现的 Bug

| Bug | 步骤 | 严重度 | 影响 |
|-----|------|:---:|------|
| C1: is_zero_lag12 前12月偏差 | S2 | 低 | 17% 训练数据特征偏差 |
| C2: MASE 分母非标准 | S13 | 低 | MASE 不可跨论文比较 |
| C3: TwoStage 与框架代码重复 | S9 | 中 | 维护风险 |
| C4: Stage2 早停验证集仅6样本 | S6 | 中 | 早停不可靠 |
| C5: Spearman 在全部81月上计算 | S2 | 低 | 轻微数据泄露 |

### D: 对论文发表的根本性结论

**1. "预测效果差"不是 Bug，是数据信息量的理论上限**

四个外生因子都是全局批次统计量，唯一物资特有的特征来自自回归（过去的自己）。系统实际上是一个 fancy 自回归模型。R² 上限由自回归成分决定：对于 CV=1.11 的物资，最佳自回归 R² ≈ 0.5-0.6。TwoStage R²=0.58 已接近此上限。

**2. 论文的核心贡献不是"高精度预测"而是"方法创新"**

论文应定位为：
- 首次将两阶段预测应用于 ECP 物资需求
- 首次系统评估批次事件因子对物资预测的贡献
- 证明外部宏观因子与单一物资需求的因果链弱
- Cross-Croston-SBA 负 R² 证明"零值不是随机间歇"

**3. 低样本下可用的模型**
