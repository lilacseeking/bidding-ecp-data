# 第一性原理诊断与改进计划

> 诊断日期: 2026-07-08 | 方法: 第一性原理思维 + 代码走查 + 数据验证

## 一、根本问题诊断

### 1.1 回到第一性原理：预测效果"差"的标准是什么？

**根本问题**: 我们到底需要多高的 R² 才能发表论文？

查阅文献（`文献对比分析.md`）发现：
- 向洪伟(2024) WOA-VMD-LSTM: MAE=0.01-0.02 → 这些值是在 **归一化后** 的数据上计算的，归一化后需求量范围 [0,1]，绝对误差极小
- 王永利(2025) PSO-ELM: 未报告 R²
- 大多数文献 **不报告 R²**，只报告归一化域下的 RMSE/MAE

**什么是"好"预测？**
- 如果 R²=0 意味着和预测均值一样好
- 如果 R²<0 意味着不如均值预测
- R²=0.5 意味着解释了 50% 的方差 → 对真实世界预测来说已经不错

**当前 TwoStage R²=0.56-0.74 → 优于均值预测 56-74% 的方差 → 对于月度物资需求预测来说是合理的**

### 1.2 代码走查：发现 3 个实际 Bug

#### Bug 1: `get_top_factors` 缓存导致 Spearman 从未执行 🔴

```python
# main() 中 (line 1534):
top4 = get_top_factors(material)       # 第一次调用: 无 df → fallback=FACTOR_NAMES[:4] → 存入缓存

# preprocess_data() 中:
top4 = get_top_factors(material, df)   # 第二次调用: 有 df → 但缓存已命中 → 返回 fallback!

# → Spearman 相关性计算从未被执行!
```

**影响**: 所有物资固定使用 `project_count, transformer_bids, monthly_bid_count, uhv_bids` 作为 top4，无论其实际相关性如何。`has_batch` 和 `digital_bids` 永远不会被选入。

**修复**: 删除 main() 中第一次无 df 的调用，让 preprocess_data 真正在训练集上执行 Spearman。

#### Bug 2: 零值信息未输入模型 🔴

当前 preprocess_data 生成的 `X_train_factors` 不包含"当月需求是否为 0"的显式特征。模型只能通过 lag1/lag12/rolling 隐式推断零值模式。

数据库事实：28/81 个月三种物资同时为零——这是结构性信息。应该作为显式特征。

**修复**: 在特征矩阵中增加 `is_zero_last_month`（上个月是否为 0）和 `is_zero_last_year`（去年同月是否为 0）。

#### Bug 3: N-HiTS 在小样本上运行 🔴

69 个训练点产生 33 个滑动窗口样本，N-HiTS 需要 3 层 MLP 堆叠。每层有独立参数池，过参数化严重。

N-HiTS 的设计论文实验数据是 **成千上万** 的时序点。在 33 个样本上运行 N-HiTS 如同用重型卡车拉一颗鸡蛋。

**证据**: N-HiTS 在所有 5 种物资上 R² < 0（包括负值 -0.002 到 -2.28），实际上不如均值预测。

**修复**: 将 N-HiTS 降级为 `max_steps=100` 的轻量模式，或在样本 < 100 时完全跳过。

### 1.3 第一性原理：数据的根本局限性

```
预测链: 外部因子 → 预测需求量

当前外部因子:
  project_count (|ρ|=0.86): 同源内源特征，不是真正的"预测"
  transformer_bids (|ρ|=0.66): 反映批次节奏
  monthly_bid_count (|ρ|=0.63): 同上
  uhv_bids (|ρ|=0.35): 特高压信号
  
理论 R² 上限 ≈ max(|ρ|²) = 0.66² = 0.44 (transformer_bids)
加上多个弱因子 → 理论 R² 上限 ≈ 0.5-0.6
加上自回归成分 (lag12) → 理论 R² 上限 ≈ 0.65-0.75

TwoStage R² = 0.56-0.74 → 已接近理论上限!
```

**根本结论**: 不是模型不好，是数据的信息量上限决定了 R² 无法超过 0.70-0.75。

## 二、改进计划

### Phase A: 修复 Bug（预计 R² 提升 0.03-0.08）

| # | Bug | 修复方式 | 预期效果 |
|---|-----|---------|---------|
| A1 | `get_top_factors` 缓存 | 删除 main() 中预调用 | 动态选择最优因子组合 |
| A2 | 零值特征缺失 | 添加 `is_zero_lag1`/`is_zero_lag12` | 所有模型受益，约 +0.02-0.05 |
| A3 | N-HiTS 样本不足 | skip when samples < 50 | 避免无意义的负 R² 结果 |

### Phase B: 数据预处理增强

| # | 增强 | 说明 |
|---|------|------|
| B1 | 零值月特征化 | 显式分离"零值 vs 非零值"信号给所有模型 |
| B2 | 近期趋势特征 | 添加 `trend_6m`: 过去 6 月简单线性回归斜率 |
| B3 | 季度模式 | 添加 `qtr_sin/qtr_cos`: 季度编码（4 季 ≠ 12 月） |

### Phase C: 评估增强

| # | 增强 | 说明 |
|---|------|------|
| C1 | 报告 MASE < 1 | 证明模型优于 Naive 基线（论文核心说服力） |
| C2 | 归一化域 R² | 同时报告归一化后的 R² 以对齐文献标准 |

## 三、执行验证

每完成一个 Phase 立即重跑 pipeline 验证 R² 变化。
