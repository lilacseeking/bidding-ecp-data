> 日期：2026-07-06
> 背景：针对 TwoStage 模型在五张预测图上的表现逐图分析失败模式，并提出替代方案

## 一、五张图逐张解读

### 图一：交流避雷器 (R²=0.51, 52/74月非零, 月均≈1,484)

**TwoStage 胜出 (+55%)。** 蓝色实际值呈现"密集但剧烈波动"的特征——每月基本都有需求，但量从几十跳到近三万。TwoStage 的 P×Q 公式在这里有效的原因：数据密度够高 (70%)，Stage 1 的分类器有足够的正负样本去区隔"有/无需求月"，Stage 2 有 42 个非零样本训练回归器。

**预测图中的失败点**：在大幅波动的月份（actual 突然从几千跳到近三万），TwoStage 和 CatBoost 都会"回归均值"——预测适中值而非捕捉极端值。

### 图二：电容式电压互感器 (R²=0.72, 52/74月, 月均≈175)

**TwoStage 显著胜出 (+41%)。** 实际值波动相对较小（对比交流避雷器），CV 更低。TwoStage 的 R² 达 0.72 说明对整体趋势的跟踪不错。

**预测图中的失败点**：可能在零值月附近预测了小幅正值——即 Stage 1 的分类器对零值月的识别不完美，概率 P 不是 0 而是 0.1~0.2，导致最终预测 ŷ = 0.15 × 回归预测 = 小正数。

### 图三：交流支柱绝缘子 (R²=0.76, 50/74月, 月均≈1,358)

**TwoStage 最佳表现 (+43%)。** 这是五个品种中模式最规律的一个——需求量级大、波动相对均匀。P×Q 在此几乎完美运作。

### 图四：断路器保护 (R²=0.17, 50/74月, 月均≈24)

**TwoStage 崩溃。CatBoost 基线 R²=0.62，TwoStage 掉到 0.17。** 这是诊断整个方法局限性的关键图表。

失败机制：月均需求仅 24——回归器从 42 个样本中学到的"预测值"约为 24，而 Stage 1 的分类器输出概率约 0.7。当 actual = 0 时，预测约 0.7×24 ≈ 17（假阳性）。当 actual = 50 时，预测仍是 ~17（回归到训练集均值）。R² = 0.62 → 0.17 的塌陷暴露了 **P×Q 公式在需求量极小时的灾难性不稳定性**——分类器的一个小误判（P 从 0 到 0.3）会在 R² 计算中产生巨大的相对误差。

### 图五：电抗器保护 (R²=0.68, 50/74月, 月均≈16)

**TwoStage 退步 (-12%)。** CatBoost 基线 R²=0.80 已经很高——说明这个序列的需求模式非常简单，单阶段回归就足够。TwoStage 的拆分为两阶段反而引入了额外的方差：Stage 1 的分类误差 × Stage 2 的回归误差 相乘放大了不确定性。当一个序列的模式足够简单时，"拆分为两阶段"是过度设计。

## 二、五张图总结：TwoStage 的三个系统性缺陷

### 缺陷 1：P×Q 乘性误差放大

TwoStage 的预测 = P × Q，其中 P 是 Stage 1 的分类概率，Q 是 Stage 2 的回归预测值。P 和 Q 的误差是**乘性叠加**的：P 偏 10% + Q 偏 10% → ŷ 偏 21%。

当 Q 本身很小（<30）时，P 的一个微小误差（比如 actual=0 但 P=0.25→预测=~6，而 actual=0）在 R² 分母中产生的相对误差远大于 Q 很大的情况（actual=0，P=0.25，预测=~350）。这解释了为什么断路器保护（月均 24）崩溃了，而支柱绝缘子（月均 1,358）表现好。

### 缺陷 2：CatBoost 在两个阶段都用了

两个阶段都用的同一个模型家族。这意味着：
- Stage 1 的分类误差和 Stage 2 的回归误差是**同源的**——它们来自相似的树结构，犯了系统性的相似错误
- 不存在"两个阶段的误差相互抵消"的可能性（如果 Stage 1 和 Stage 2 用了完全不同原理的模型，反而可能互相纠正）

### 缺陷 3：分类器和回归器独立训练

这是 TwoStage 文档自己承认的局限性（第 7 节）。但它的后果比"不是最优"更严重：分类器和回归器的目标函数（Logloss vs RMSE）在大需求和小需求上的相对权重不同。当分类器更关注高频出现的零值，而回归器更关注少量的大额需求时，P×Q 把两者的训练偏差乘到了一起。

## 三、基于数据特征的模型推荐

### 核心约束回顾

| 特征 | 数值 | 对模型的硬约束 |
|------|:--:|------|
| 个体物资密度 | 12-20% | 80%+ 的月份为零 — VMD/LSTM 不可行 |
| 训练集非零样本 | 8-13 个月 | 任何 >10 参数的 ML 模型必然过拟合 |
| 测试集非零事件 | 1-3 个月 | R² 主要由 1-3 个数据点决定 — 极不稳定 |
| 需求规模跨度 | 几十~几百万 | 不同物资的绝对量级差 4-5 个数量级 |

### 模型一：Croston + TSB 框架 + LightGBM (★★★★★)

**为什么是它**

Croston 方法 (1972) 是间歇性需求预测的奠基性框架。核心思想：

```
不预测月度需求量 → 预测两个独立的随机变量：
  1. 需求间隔 (demand interval): 两次需求之间相隔几个月？
  2. 需求量 (demand size): 发生需求时，量是多少？

当前月份的预测 = size_forecast / interval_forecast
```

**50 年来 Croston 及其变体（SBA、TSB）全是用指数平滑做参数估计的。没有任何已发表论文用梯度提升树替代过指数平滑。** 这就是创新点。

**具体实现（约 150 行代码）**

```python
from sktime.forecasting.tsb import TSB

# Step 1: 从训练数据提取 interval 和 size 序列
intervals = []  # e.g., [3, 2, 5, 1, 7, ...] — 每次需求间隔的月数
sizes = []      # e.g., [86887, 37490, 240000, ...] — 每次非零需求量

# Step 2: TSB 基准预测 (传统指数平滑)
tsb_forecast = tsb_model.predict(fh=np.arange(1, 13))

# Step 3: LightGBM 对 interval 和 size 独立建模
# 特征: month_sin/cos, year_index, last_interval, avg_interval, last_size, avg_size
lgb_interval = LightGBMRegressor()
lgb_interval.fit(X_interval_features, intervals)

lgb_size = LightGBMRegressor()
lgb_size.fit(X_size_features, sizes)

# Step 4: 预测 + 概率衰减 (TSB 模式)
interval_pred = lgb_interval.predict(X_test)
size_pred = lgb_size.predict(X_test)
prob_pred = 1.0 / interval_pred  # 本月发生需求的概率

# 如果距离上次需求已超过历史最长间隔的 2× → 概率指数衰减
if months_since_last > 2 * np.mean(intervals):
    prob_pred *= 0.5 ** (months_since_last / np.mean(intervals))

final_forecast = prob_pred * np.maximum(size_pred, 0)
```

**为什么比 TwoStage 好**

| 维度 | TwoStage | Croston+TSB+LightGBM |
|------|:--|:--|
| 零值处理 | Stage 1 分类(80%被零值淹没) | 间隔变量天然编码零值 |
| 样本效率 | 每个零值月都是"负样本"——浪费 | 只关注非零月——每个样本都有信息 |
| 误差来源 | P×Q 乘性叠加 | interval 和 size 独立预测后除法——除性解耦 |
| 可解释性 | 黑箱概率 | "预计每 X 个月发生一次需求，每次约 Y" |
| 发表风险 | Jiang et al. (2019) 已做过 | 文献中无人用 LightGBM 替代 Croston 的指数平滑 |

**短板**
- 需要 ≥5 个需求事件才能稳定估计 interval（当前每种物资 ≥12 个，足够）
- 假设 interval 和 size 独立（不总是成立，但比 TwoStage 的"有/无"vs"量"独立假设更合理）
- 对突然的结构性变化（如 2025 年需求翻倍）响应慢

**文献支持**

- Croston (1972, *Operational Research Quarterly*): 间歇性需求预测奠基文，引文 1,200+
- TSB: Teunter, Syntetos & Babai (2011, *EJOR*): 加入需求概率衰减，引文 500+
- Sanguri et al. (2024, *IJPR*): Croston 族方法在备件预测上的最新验证

### 模型二：LightGBM Direct Multi-Step (★★★★★)

**为什么是它**

M5 预测竞赛（2020，全球 5,000+ 团队参与）的 Accuracy 冠军是 LightGBM 集成而非任何神经网络。后续研究（Kapetanovic et al., 2025, arXiv:2506.05941）对 70% 间歇性的零售数据确认：**LightGBM/XGBoost 持续超越 N-BEATS、N-HiTS 和 TFT**。关键原因：

- 树模型不需要序列连续性——每行训练数据是独立采样
- 零值和极端值都不会影响梯度（分裂点是基于顺序的，不是基于距离的）
- leaf-wise 生长（LightGBM）比 level-wise（XGBoost）和 symmetric（CatBoost）在间歇性数据上更灵活

**具体实现（约 200 行代码）**

```python
# 特征工程 (约 22 维):
features = {
    # 时间编码
    'month_sin': sin(2π × month / 12),
    'month_cos': cos(2π × month / 12),
    'year_index': (year - 2020),
    
    # 自回归滞后
    'demand_lag1': demand_series[-1],
    'demand_lag2': demand_series[-2],
    'demand_lag3': demand_series[-3],
    'demand_lag6': demand_series[-6],
    'demand_lag12': demand_series[-12],
    
    # 滚动统计
    'avg_demand_3m': mean(demand_series[-3:]),
    'avg_demand_6m': mean(demand_series[-6:]),
    'avg_demand_12m': mean(demand_series[-12:]),
    'max_demand_12m': max(demand_series[-12:]),
    
    # 需求间隔特征 (Croston-inspired)
    'months_since_last_demand': months since last non-zero,
    'is_in_zero_streak': months_since_last_demand > 3,
    'avg_interval': mean(historical_intervals),
    'avg_size': mean(historical_nonzero_sizes),
    
    # 批次节奏
    'is_march': month == 3,
    'is_may': month == 5, 
    'is_september': month == 9,
    
    # 外部因子 (保留 2-3 个相关性最高的)
    'copper_price_3m_ago': copper_price[-3],
    'pmi': pmi,
}

# 12 个独立 LightGBM 模型, h=1 到 12
for h in range(1, 13):
    model_h = LightGBMRegressor(
        n_estimators=200,  # 小样本, 少量树防止过拟合
        max_depth=3,        # 浅树
        min_child_samples=5, # 叶子最少样本
        reg_alpha=0.5,      # L1 正则化
        reg_lambda=1.0,     # L2 正则化
    )
    # 训练: (X_train, y_train_shift_h) 
    model_h.fit(X, y[t+h])
```

**短板**

- 没有预测区间（除非加 quantile regression）
- 需要约 22 维特征工程（但一次做完可复用）
- 对需求模式的结构性变化不敏感（和 CatBoost 一样）

### 模型三：LightGBM + TSB 加权 TwoStage (★★★☆, 保守过渡方案)

**在现有 TwoStage 框架上做四点最小改动**:

1. Stage 1: CatBoostClassifier → **LightGBMClassifier** + `scale_pos_weight = n_neg/n_pos` + `eval_metric='average_precision'`
2. Stage 2: CatBoostRegressor → **LightGBMRegressor** + `objective='tweedie'` (零膨胀友好分布)
3. 最终预测: `ŷ = P × max(0, Q)` 不变, 但加入 TSB 框架的**需求概率衰减**机制
4. 样本不足时的退路: 两阶段退化为 LightGBM Direct Single-Stage（详见模型二）

**优势**: 改动最小 (<30 行), 保留现有 TwoStage 论文结构

**短板**: 本质框架未变 — P×Q 的乘性误差放大对月均 <30 的物资仍然存在

### 模型四：Count Time Series 模型 — ZIPoINAR(1) (★★★, 统计学方法)

**来自帕多瓦大学 2025 年博士论文的验证**:

ZIPoINAR(1) (Zero-Inflated Poisson Integer Autoregressive) 是**专门为零膨胀整数序列设计的统计模型**。在间歇性备件需求测试中：
- 比 Croston **提升 20%** (sCE 指标)
- 比 Croston **提升 23%** (RMSSE 指标)
- 比 Croston **提升 8-9%** (MASE/sAPIS 指标)

优势: 数学上精确建模了"零膨胀"分布 — 假设数据生成过程是 compound Poisson

短板: 模型假设计数数据 (整数)，需要验证物资需求量是否满足此假设；Python 实现不如 LightGBM 成熟

**文献**: Rafieisangari P. Master's Thesis, University of Padua, 2025.
