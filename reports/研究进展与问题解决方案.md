# 电力物资需求量预测 — 研究进展、问题诊断与解决方案

## 一、研究目标

撰写一篇可发表在中文核心/SCI期刊上的小论文，选题方向为 **"基于多源数据融合的电力物资需求量预测方法"**。

**论文基本要素：**

| 要素 | 内容 |
|------|------|
| 研究对象 | 国家电网物资采购需求量（月度频率） |
| 数据来源 | 国网新一代电子商务平台 ECP2.0 (ecp.sgcc.com.cn) |
| 数据范围 | 2019-11 ~ 2026-07，81个月 |
| 目标变量 | 单种物资的月度需求数量 |
| 影响因子 | ~~市场价格因子（铜/铝期货）、工业用电量、项目数量、日历编码（公历+农历）~~ → 经实验验证后仅保留项目数量 + 批次事件特征。铜铝期货/工业用电/农历编码已删除(|ρ|<0.26) |
| 候选模型 | CatBoost, VMD-CatBoost, VMD-Transformer-CatBoost, VMD-SVR 等 |

## 三、当前面临的两个核心困难

### 困难1：数据选择困境

**冀北公司数据 (已放弃)**

| 问题 | 详情 |
|------|------|
| 数据量少 | 仅 492 条公告，97 条物资正刊，40,773 条明细 |
| 零值过多 | 单种物资在 81 个月中可能只有 20-30 个月有数据 |
| 模型崩溃 | 训练集有效样本过少，VMD 无法对大量连续零值序列进行有意义的模态分解 |
| 结论 | 冀北数据**不适合做物资级的需求预测**，仅适合做"公告数量"级的时间序列分析 |

**国网总公司数据 (当前)**

| 优势 | 问题 |
|------|------|
| 数据量大 (928公告, 36万+明细) | 单种物资仍有 29-31 个零值月 (占36-38%) |
| 时间跨度充足 (81个月) | 零值分布不规则，非季节性零值 |
| 覆盖全品类物资 | 外部因子 (铜/铝价、用电量) 与物资需求的直接因果关联弱 |

**冀北 vs 国网总部 关键对比**

```
                    冀北电力              国网总部
公告总数             497                   928
物资明细             40,773                366,101
单物资有数据月       20-30月               50-52月/81
零值占比             60-75%                36-38%
模型训练             VMD崩溃              可训练但外部因子解释力弱
适用方向             公告数量预测           物资级需求预测 ✅
```

### 困难2：VMD-SVR 模型拟合优度异常（数据泄露问题）

#### 2.1 现象

VMD-SVR 在三种物资上的 R² 均超过 0.98，而其他 4 种模型 R² 最高仅 0.52。这一巨大差距**不能用模型性能差异解释**。

#### 2.2 根因分析：VMD-SVR 的数据泄露机制

VMD 的数学性质是 `Σ IMF_k(t) ≈ y(t)`（分解-重构恒等式）。当前架构将 IMF 作为输入特征：

```
错误流程:
  Step 1: VMD(y_train) → [IMF_1, IMF_2, IMF_3]  ← IMF是y的分解产物
  Step 2: SVR.fit([IMF_1, IMF_2, IMF_3, 外部因子], y_train)
          ↑ SVR学习"三个分量求和 ≈ y"，这是数学恒等式
  Step 3: IMF_test = seasonal_naive(IMF_train)  ← "复制去年同期模式"
  Step 4: SVR.predict([IMF_test, 外部因子]) → ŷ_test
          ↑ "预测" ≈ 去年同期的需求模式，而非基于外部因子推理
  Step 5: R² ≈ 0.99  ← 高R²来自复制去年数据，不是真正的预测能力
```

**可视化理解：**

```
┌─────────────────────────────────────────────────────────────┐
│  我们以为 VMD-SVR 在做的事:                                   │
│                                                             │
│  外部因子(铜价/铝价/用电量) → SVR → 预测需求                  │
│                                                             │
│  实际 VMD-SVR 在做的事:                                      │
│                                                             │
│  y_train → 分解为 IMF₁, IMF₂, IMF₃                          │
│  IMF₁ ≈ 去年模式的高频波动                                   │
│  IMF₂ ≈ 去年模式的中频周期                                   │
│  IMF₃ ≈ 去年模式的低频趋势                                   │
│                                                             │
│  "预测" = 把去年同期的三个分量重新加总 ≈ 去年的需求值          │
│  外部因子(铜价/铝价)几乎零贡献！                              │
└─────────────────────────────────────────────────────────────┘
```

#### 2.3 证据

| 证据 | 说明 |
|------|------|
| CatBoost baseline R² 仅 0.15~0.49 | 只用外部因子预测，效果差 |
| VMD-SVR R² = 0.98~0.99 | 加入 IMF 特征后飙升至近乎完美 |
| VMD-Transformer 直接求和 R² < 0 | Transformer 预测 IMF 有误差，求和后误差放大 |
| VMD-CatBoost 同样用 IMF 但 R² 远低于 SVR | 树模型学习 "求和" 比核方法困难 |

**结论：VMD-SVR 的高 R² = IMF 重构误差 + 季节性朴素预测精度，不代表模型优秀。**

#### 2.4 关键洞察：正确架构 vs 错误架构

```
❌ 错误架构 (当前 VMD-SVR):
   y_train → VMD → IMF_train → [作为特征输入模型] + 外部因子 → predict y
   问题: IMF 包含 y 的信息，等于把答案告诉模型再让它猜

✅ 正确架构:
   外部因子 → model_1 → 预测 IMF_1(测试期)
   外部因子 → model_2 → 预测 IMF_2(测试期)
   外部因子 → model_3 → 预测 IMF_3(测试期)
   Σ 预测的 IMF_k → 最终预测 ŷ
   挑战: 外部因子预测力弱 → 每个 IMF 都预测不准 → 求和更不准
```

这个架构修正后，VMD 方法的效果会大幅下降（因为外部因子难以预测 IMF），但这才是**真实的预测能力**。

**从逻辑上来说，不应该把VMD分解放在前面。因为VMD分解序列后，每个本征模态函数IMF的含义、物理意义不尽相同，且难以解释。比较好的方案是，先用其他模型做一次初步预测，然后将初步预测后的结果 与 真实值 进行差分，得到残差分量，再对残差分量进行模态分解，最后预测，这样或许合理一些。**

## 四、解决方案

### 4.1 立即修复：删除有问题的 VMD 架构

当前 VMD-CatBoost、VMD-Transformer-CatBoost、VMD-SVR、VMD-Transformer 四个模型的**共同问题**是：

```
IMF 作为特征 = 数据泄露
季节朴素外推 = 伪精度
```

**所有使用 IMF 作为输入特征的架构都应该废弃**，因为：
- 它们不是在做预测，是在做"IMF重构 + 去年模式复制"
- 论文审稿人能够识别这种数据泄露
- 发表后可能面临撤稿风险

### 4.2 正确架构：残差 VMD 预测

```
Step 1: 外部因子 → Base Model → 初步预测 ŷ_base
Step 2: 残差 r = y_train - ŷ_base_train
Step 3: VMD(r_train) → K个残差IMF
Step 4: 外部因子 → model_k → 预测每个残差IMF_k_test
Step 5: 最终预测 = ŷ_base_test + Σ 残差IMF_k_test
```

这个架构的优势：
- Base Model 先从外部因子给出合理的基础预测
- VMD 只在残差上运行，分解的是"基础模型解释不了的部分"
- 残差中的可预测结构（如季节模式未完全被外部因子捕捉）能被提取
- 没有数据泄露：残差 IMF 不包含 y 本身

### 4.3 增强特征工程

#### 短期（立即可做）：批次事件特征

ECP 平台的批次预安排是公开信息。可以在特征工程中增加：

| 特征 | 说明 | 数据来源 |
|------|------|---------|
| `is_bid_batch_month` | 本月是否有招标批次发布 | ECP 批次预安排 |
| `batch_type_onehot` | 批次类型 (输变电/特高压/电源/数字化) | ECP |
| `cumulative_batches` | 本年度累计批次数 | 计算 |
| `batch_phase` | 本年度第几轮批次 | 计算 |
| `days_since_last_batch` | 距上次批次天数 | 计算 |
| `monthly_notice_count` | 当月国网总部公告总数 | ECP DB (已有) |
| `monthly_bid_amount` | 当月中标金额汇总 | ECP DB 中标公告 |

**预期效果：CatBoost baseline R² 从 0.15~0.49 提升到 0.50~0.70**

#### 中期（1-2周）：宏观投资因子

| 特征 | 说明 | 采集方式 |
|------|------|---------|
| `monthly_grid_invest` | 全国电网基建投资月度完成额 | 中电联月报 |
| `annual_invest_plan` | 国网年度投资计划 | 年度工作会议通稿 |
| `invest_completion_rate` | 投资完成进度 (实际/计划) | 计算 |

#### 已在特征中但可能未充分利用的因子：

- `notice_count`（项目数量）——当前 top-1 因子，Spearman 相关性最高
- `lunar_month_sin/cos` —— 农历月份编码（在特征矩阵中，但 Spearman 排名未进 top4）
- `lag12` —— 去年同期需求（已有，但可能被 CatBoost 欠拟合）

## 五、建议的实验方案

### 实验设计

```
数据集: 国网总部 201911-202607, 81个月, 3种物资

数据划分: 69/12 (Train/Test), 可额外做 rolling window 交叉验证

Baseline 模型 (必须包含):
  1. Naive Seasonal (每年同月 = 去年同月, 作为 naive 下界)
  2. CatBoost + 当前13维特征 (已有 R² = 0.15-0.49)
  3. SARIMA (经典统计方法)

实验组 A: 增强特征 + CatBoost
  - CatBoost + 批次事件特征 (预期 R² ≈ 0.50-0.70)
  - CatBoost + 批次事件 + 宏观投资 (预期 R² ≈ 0.55-0.75)

实验组 B: 残差 VMD 方法
  - CatBoost(A) 初步预测 → 残差 VMD → 各 IMF SVR → 求和 + 初预 (预期 R² ≈ 0.55-0.78)

实验组 C: 消融实验
  - 去掉批次特征 (验证批次特征的贡献)
  - 去掉农历编码 (验证农历因子的贡献)
  - 去掉市场因子 (验证铜/铝价格的贡献)
```

### 评估指标

| 指标 | 用途 |
|------|------|
| RMSE | 主指标，量纲可解释 |
| MAE | 鲁棒性指标 |
| R² | 论文标准指标 |
| SMAPE | 处理零值月份的相对误差 |
| MDA (Mean Directional Accuracy) | 预测方向的正确率 |
