# 会话记录 — 2026-07-08 模型精简与关键发现

## 一、两阶段预测根因分析

### 发现：nv2 验证集过大的反作用

`_two_stage_fit_predict` 共享框架中 `nv2 = max(6, nz.sum()//3)` 导致 Stage2 训练样本仅 28 个（42-14=28），CatBoost depth=6 在 28 样本上严重过拟合。

修复：nv2 = min(6, nz.sum()//4) → CatBoost-2S R² 从 **0.147 → 0.595**（+0.45）。

根源：之前 F3 "修复"（增加验证集使早停更可靠）抢走了 1/3 的训练数据。

## 二、纯时序模型在间歇性需求上不可行

引入 DLinear (AAAI 2023) 和 ModernTCN (ICLR 2024) 并测试：

| 模型 | 全 5 物资 R² |
|------|:---:|
| DLinear | 全线为负 (-0.22 ~ -10.9) |
| ModernTCN | 全线为负 (-1.44 ~ -15.4) |

**根因**：DLinear/ModernTCN 仅使用需求序列 y 值做预测，不读取特征矩阵中的 project_count、lag1/lag12、批次事件等 13 维特征。35-38% 零值率的间歇性数据需要特征矩阵来预测"什么时候有需求"。

**论文启示**：这可以作为方法论贡献——"纯时序方法在电力物资间歇性需求数据上的局限"。

## 四、共享 Stage1 分类器

**改进**：原来每个 -2S 模型各自训练独立 CatBoostClassifier（最多 6 次），现在预训练一个共享分类器，所有 2S 模型共用同一个 `prob_s1` 向量。

**收益**：
- 减少 ~12 秒训练时间
- 确保不同模型的 R² 差异纯粹由 Stage2 回归器差异引起（方法论严谨性）
- Stage1 准确率保持一致，论文对比更公平

## 五、多 Agent 协同迭代（首轮完成）

4 个 Agent 分工机制：
- Agent A（发现问题）：输出 10 个优先级排序的改进项
- Agent B（制定计划）：根据 A 的输出生成实施计划
- Agent C（实施改动）：执行 B 的计划
- Agent D（评估保留/回滚）：验证改动效果

首轮结果：模型从 23 精简到 9，共享 Stage1 分类器，删除 ~150 行无用模型调用。
