# 文档管理体系实施方案

## 目标

解决当前三个核心问题：
1. **数量膨胀**：22 个 md 文件散落在 reports/ 下，无分类
2. **冗余未清理**：3 篇审核报告属临时文档，应合并到主文档
3. **无统一入口**：没有导航首页，无法快速定位

## 方案设计

### 1. 目录结构重组

```
bidding-ecp-data/
  docs/                          ← MkDocs 文档根目录
    index.md                     ← 导航首页
    数据层/
      数据合理性验证.md
      数据选择分析.md
      数据稀疏度与合并方案.md
    方法层/
      影响因子可行性验证报告.md
      影响因子增强方案.md
      模型合理性评估.md
      大类vs品类相关性分析.md
    工程层/
      项目逻辑步骤拆解.md
      代码资产价值评估.md
      步骤确定性评估.md
      两阶段预测模型说明.md
    决策层/
      最终执行方案.md
      国网投资计划影响因子可行性分析.md
      电力物资需求预测创新模型方案.md
      研究进展与问题解决方案.md
    文献/
      基于VMD-LSTM-LightGBM的..._张未.md      ← 从 book/ 移入
      ... (5 篇参考文献)
    早期记录/                     ← 归档，不出现在导航中
      审核报告_R1.md
      审核报告_R2.md
      审核报告_R3.md
      研究方案讨论纪要.md
      数据获取方案.md
      verification_report.md
```

### 2. 文档处理决策

| 文件 | 处理方式 | 理由 |
|------|---------|------|
| 审核报告_R1/R2/R3 | 归档到早期记录 | 审核已完成，结论已合并到主文档 |
| 研究方案讨论纪要 | 归档 | 会议记录，非正式分析 |
| 数据获取方案 | 归档 | 内容已被"数据合理性验证"覆盖 |
| verification_report | 归档 | 早期验证报告，信息已过期 |
| book/ 下 5 篇参考文献 | 移到 docs/文献/ | 与 reports/ 统一管理 |

### 3. 导航首页 (index.md)

提供：
- 项目概述（2 句）
- 文档地图（按 4 层分类列出，每篇一句话摘要）
- 快速入口（论文作者看这里 / 开发者看这里）

### 4. 新增 `.doc-links.yaml` 漂移检查配置

手工维护关键文档与代码文件的映射关系。CI 或手动脚本可检查代码是否有未被文档覆盖的大改动。

格式：
```yaml
- doc: docs/工程层/项目逻辑步骤拆解.md
  codes:
    - src/pipeline.py
    - vmd-catboost/main.py
  sections:
    - S3 Excel解析入库
    - S8 特征工程
```

## 执行步骤

| 步骤 | 操作 |
|------|------|
| 1 | 安装 mkdocs-material |
| 2 | 创建 docs/ 目录及子目录 |
| 3 | 移动文件到对应目录，归档临时文档 |
| 4 | 移动 book/ 下的参考文献到 docs/文献/ |
| 5 | 编写 index.md 导航首页 |
| 6 | 创建 mkdocs.yml 配置 |
| 7 | 创建 .doc-links.yaml 漂移配置 |
| 8 | mkdocs build 验证 |
| 9 | 复核：检查导航完整性、文件正确性 |

- [ ] 文档站点有搜索功能
- [ ] 目录结构清晰，4 层分类命名准确
