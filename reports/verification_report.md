## 验证7：列表API字段完整性

### 问题
列表API是否直接包含"物资名称"和"需求数量"？

### 验证方法
检查 noteList 返回的每条记录的所有字段。

### 验证结果

**已有字段（10个）**：
| 字段 | 说明 | 可用性 |
|------|------|--------|
| id/noticeId | 公告ID | ✅ |
| title | 公告标题 | ✅ |
| code | 项目编号 | ✅ |
| publishOrgName | 发布单位 | ✅ |
| orgId | 单位ID | ✅ |
| noticePublishTime | 发布时间 | ✅ |
| noticeType | 公告类型编码 | ✅ |
| doctype | 文档类型 | ✅ |
| firstPageDocId | 详情文档ID | ✅ |
| topBeginTime/topEndTime | 投标起止时间 | ✅ |

**缺失字段**：
| 字段 | 可用性 |
|------|--------|
| 物资名称 | ❌ |
| 物资描述/规格 | ❌ |
| 需求数量 | ❌ |
| 计量单位 | ❌ |
| 包号 | ❌ |
| 限价/预算金额 | ❌ |
| 中标企业名称 | ❌ |
| 中标金额 | ❌ |

### 结论
```
⚠️ LIMITED — 列表API不含任何物资级别的数据字段。
物资名称和需求数量必须从详情页/货物清单附件中获取。
```

## 风险与限制

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| ECP平台改版导致API变更 | 中 | 定期检测API可用性，版本号监控 |
| 详情页反爬机制 | 中 | 使用Playwright模拟真实浏览器，控制频率 |
| 附件格式不统一 | 低 | 支持多种解析器（Excel/ZIP/PDF） |
| IP被封禁 | 低 | 控制请求频率（>0.5s间隔），使用代理池 |
| 中标公告菜单ID未知 | 中 | 需进一步逆向SPA路由 |
| 仅5-6年历史数据 | 中 | 可接受（平台限制，非技术问题） |
